    **Step 2: node_builder.rs** — Per D-09, using the IntoLayer trait pattern:

    `IntoLayer` trait — enables closures to return either `LayerBuilder` or `CubeBuilder`:
    ```rust
    pub trait IntoLayer {
        fn into_layer(self) -> LayerBuilder;
    }
    impl IntoLayer for LayerBuilder { fn into_layer(self) -> LayerBuilder { self } }
    impl IntoLayer for CubeBuilder { fn into_layer(self) -> LayerBuilder { /* push node to parent, return parent */ } }
    ```

    This enables the D-08 syntax: `|l| l.cube().position(v)` where the closure returns `CubeBuilder` which auto-converts via IntoLayer. Also handles multiple children: `|l| l.cube().position(v).cube().rotation(r)` — each `.cube()` call on CubeBuilder finalizes the current cube (pushes to parent) and starts a new one.

    `LayerBuilder`:
    - Internal: `z_order: i16`, `children: Vec<SceneNode>`, `pipeline: Option<Pipeline>`.
    - `pub fn cube(self) -> CubeBuilder` — creates CubeBuilder with `parent: self`.
    - `fn into_node(self) -> SceneNode` — produces SceneNode with `NodeKind::Layer { z_order }`.

    `CubeBuilder`:
    - Internal: `parent: LayerBuilder`, `node: SceneNode` (in-progress cube).
    - `pub fn position(mut self, v: impl Into<Vec3>) -> Self` — sets node.transform.position.
    - `pub fn rotation(mut self, sig: &Signal<f32>) -> Self` — inserts `PropValue::Reactive` into node.props["rotation"]. Per D-09, signal props accept `&Signal<T>`.
    - `pub fn scale(mut self, v: impl Into<Vec3>) -> Self` — sets node.transform.scale.
    - `pub fn cube(self) -> CubeBuilder` — finalize current cube (push to parent.children), create new CubeBuilder with same parent.
    - `impl IntoLayer for CubeBuilder` — push node to parent.children, return parent.

    `GroupBuilder`: similar pattern, wraps children under NodeKind::Group.
