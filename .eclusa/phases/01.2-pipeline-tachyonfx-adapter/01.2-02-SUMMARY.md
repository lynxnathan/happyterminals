## Decisions Made
- Wrapped `tachyonfx::Effect` (struct) rather than generic `S: Shader` because tachyonfx::fx::* functions return `Effect` (which wraps `Box<dyn Shader>` internally), not `impl Shader`. This is a deviation from the plan's generic signature but matches the actual API.
- Duration conversion happens in both the adapter (for process calls) and convenience constructors (for tachyonfx::fx::* factory calls). This double-point conversion ensures the tachyonfx custom Duration type is used throughout.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] Changed TachyonAdapter from generic to concrete**
- **Found during:** Task 1 (TachyonAdapter implementation)
- **Issue:** Plan specified `TachyonAdapter<S: Shader>` but tachyonfx::fx::* functions return `tachyonfx::Effect` (a struct wrapping `Box<dyn Shader>`), not `impl Shader`. The generic design would not work with the actual API.
- **Fix:** Made `TachyonAdapter` a concrete struct wrapping `tachyonfx::Effect` (aliased as `Fx` in our crate)
- **Files modified:** crates/happyterminals-pipeline/src/adapter.rs
- **Verification:** All tests pass, all convenience constructors compile
