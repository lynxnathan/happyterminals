## Implementation Decisions (locked)

### Effect trait (PIPE-01, PIPE-02)
- **D-01:** `Effect` trait signature: `fn apply(&mut self, grid: &mut Grid, dt: Duration) -> EffectState` where `EffectState` is `Running | Done`. Effects are stateful (own timers, progress counters).
- **D-02:** Our trait keeps the name `Effect`. `tachyonfx::Effect` is re-exported as `Fx` in our public surface. This resolution is committed to code + CHANGELOG before any Pipeline consumer code.
- **D-03:** `Pipeline = Vec<Box<dyn Effect>>` â€” trait objects, not generic chains. Runtime-constructable for JSON recipes (Phase 3.2) and Python (Phase 4.x).

### TachyonAdapter (PIPE-03, PIPE-04)
- **D-04:** `TachyonAdapter<S: tachyonfx::Shader>` wraps a tachyonfx effect as one of our `Effect` trait objects. Real `Duration` dt is forwarded to tachyonfx's timing system.
- **D-05:** [auto] tachyonfx uses `ratatui::buffer::Buffer` directly for its effects. Since our Grid derefs to Buffer, the adapter can pass `grid.deref_mut()` (via inner_mut()) to tachyonfx. The adapter translates our `dt: Duration` into tachyonfx's frame-timing model (tachyonfx effects track elapsed time internally via their `process()` method).

### Pipeline execution (PIPE-05, PIPE-06)
- **D-06:** [auto] Pipeline runs effects in sequence (Vec order). Each effect's `apply()` receives the Grid after previous effects have mutated it. This is the simple, composable model â€” parallel effect composition is not needed for M1.
- **D-07:** [auto] When an effect returns `EffectState::Done`, it stays in the Vec but is skipped on subsequent frames (a `done` flag or checked via a method). Pipeline itself returns `Done` when all effects are `Done`. Effects can be reset via a `reset()` method on the trait for looping/reuse.
- **D-08:** Whole-grid composition only. No per-object effect loops. The Pipeline applies each effect to the entire Grid. Object-specific effects are achieved by composing with clip regions in Phase 1.4+.

### Quality (PIPE-07, DSL-03)
- **D-09:** All errors on public surface are `Result`-based. `clippy::unwrap_used` and `clippy::expect_used` denied crate-wide.
- **D-10:** Invariant: effects mutate Grid only through Pipeline::apply(). Documented in public docstrings. Where possible, enforced by making Grid only accessible via `&mut Grid` in the `apply()` signature (no global Grid access).
- **D-11:** Criterion benchmark proves `Pipeline::run_frame()` overhead is O(cells), not O(objects Ă— effects). Benchmark runs the 10 wired effects on a 200Ă—60 Grid for 60 frames.
