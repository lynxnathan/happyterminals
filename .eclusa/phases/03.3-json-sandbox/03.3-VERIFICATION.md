**Phase Goal:** JSON recipes execute in a security sandbox where effect resolution is static, file paths are constrained, and user-provided strings cannot inject ANSI escapes.

## Goal Achievement

### Observable Truths

| # | Truth | Status | Evidence |
| --- | ----- | ------ | -------- |
| 1 | An unregistered effect name in a JSON recipe returns a RecipeError before any scene construction | VERIFIED | `EffectRegistry::resolve` at `sandbox.rs:63-71` returns `RecipeError::UnknownEffect { name }` when name is not in registry. Test `sandbox_registry_resolve_unknown_returns_error_with_name` (sandbox.rs:263-274) asserts both the error variant and that the offending name appears in the message. `SandboxConfig` carries the registry into `load_recipe_sandboxed` so the error path is wired end-to-end even though the current recipe schema has no effect-name field yet. |
| 2 | A mesh path containing '..' or an absolute path is rejected before file I/O | VERIFIED | `sanitize_path` (sandbox.rs:117-154) is pure-string; no `canonicalize()` call. Rejects `..`, `.`, leading `/`, leading `\`, Windows drive letters, empty strings, and backslash-smuggled traversal. `load_recipe_sandboxed` calls `sandbox_recipe_node` → `sanitize_path` before `SceneIr` construction (json.rs:393-400). Tests `sandbox_rejects_path_traversal`, `sandbox_rejects_absolute_mesh_path`, `sandbox_rejects_traversal_inside_group` all pass. |
| 3 | A user-provided string containing ANSI escape sequences renders as literal characters in the SceneIr | VERIFIED | `strip_ansi` (sandbox.rs:173-243) is a byte-level scanner covering CSI, OSC, and simple ESC sequences with UTF-8 safety. Applied by `sandbox_props` / `sandbox_json_value` to every string in props (including nested arrays/objects). Test `sandbox_strips_ansi_from_props` asserts `"\x1b[31mred\x1b[0m"` lands as `"red"` in `SceneIr`; `sandbox_strips_ansi_from_nested_prop_array` covers nested-array case. |
| 4 | Registered built-in effects (dissolve, slide-left, fade-to-black) resolve without error | VERIFIED | `EffectRegistry::default()` (sandbox.rs:86-96) registers exactly those three names. Test `sandbox_default_registry_contains_three_builtins` (sandbox.rs:254-260) asserts `len() == 3` and all three resolve `Ok(())`. |
| 5 | A mesh path within the asset root resolves successfully | VERIFIED | `sanitize_path("bunny.obj", root)` returns `Ok(root.join("bunny.obj"))`. Test `sandbox_accepts_valid_mesh_path` (json.rs:1272-1281) loads a recipe with mesh path `bunny.obj` through `load_recipe_sandboxed` and confirms the stored prop path is `"bunny.obj"`. `sandbox_path_accepts_subdir` confirms sub-directory paths also pass. |
