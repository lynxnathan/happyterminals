<objective>
Add security sandboxing to the JSON recipe loader: a static effect registry that rejects unknown effect names, path sandboxing that constrains mesh file paths to a configurable asset root, and ANSI-escape stripping on user-provided strings.

Purpose: Prevents untrusted JSON recipes from accessing arbitrary files, injecting terminal escapes, or triggering dynamic code loading. Required for safe JSON recipe execution (v2.0 exit gate 3).
Output: sandbox.rs with three utilities, load_recipe_sandboxed() that applies all three, updated prelude.
</objective>

  <behavior>
    - EffectRegistry::default() contains entries: "dissolve", "slide-left", "fade-to-black"
    - EffectRegistry::resolve("dissolve") returns Ok(())
    - EffectRegistry::resolve("nonexistent") returns Err with effect name in message
    - EffectRegistry::register("custom") then resolve("custom") returns Ok(())
    - sanitize_path("bunny.obj", "/assets") returns Ok(PathBuf) pointing inside /assets
    - sanitize_path("../etc/passwd", "/assets") returns Err
    - sanitize_path("subdir/../bunny.obj", "/assets") returns Err (any .. component rejected)
    - sanitize_path("/etc/passwd", "/assets") returns Err (absolute paths rejected)
    - sanitize_path("subdir/model.obj", "/assets") returns Ok
    - strip_ansi("hello") returns "hello" unchanged
    - strip_ansi("\x1b[2Jhello") returns "hello" (CSI sequence removed)
    - strip_ansi("\x1b[31mred\x1b[0m") returns "red" (SGR sequences removed)
    - strip_ansi("a\x1b]0;title\x07b") returns "ab" (OSC sequence removed)
    - strip_ansi("\x1b[?25h") returns "" (private mode sequence removed)
  </behavior>
  <action>
    Create `crates/happyterminals-dsl/src/sandbox.rs` with three components:

    1. **EffectRegistry** -- a struct wrapping `HashSet<String>`. 
       - `Default` impl populates with "dissolve", "slide-left", "fade-to-black" (matching the three built-in TransitionEffect structs from happyterminals-scene).
       - `register(&mut self, name: &str)` adds a name.
       - `resolve(&self, name: &str) -> Result<(), RecipeError>` returns `Ok(())` if found, or a new `RecipeError::UnknownEffect { name }` variant if not.

    2. **sanitize_path(relative: &str, asset_root: &Path) -> Result<PathBuf, RecipeError>**
       - Reject if `relative` is empty.
       - Reject if `relative` starts with `/` or contains a Windows drive letter prefix.
       - Split on `/` and reject if ANY component is `..` or `.`.
       - Return `asset_root.join(relative)` on success.
       - Error variant: `RecipeError::PathTraversal { path: String }`.
       - Do NOT call `canonicalize()` -- that requires the file to exist. Pure string validation.

    3. **strip_ansi(input: &str) -> String**
       - Remove ANSI escape sequences using a simple state machine or regex-free byte scan:
         - CSI sequences: `\x1b[` followed by params and a final byte in `@`..`~`
         - OSC sequences: `\x1b]` followed by content until BEL (`\x07`) or ST (`\x1b\\`)
         - Simple escapes: `\x1b` followed by a single byte in `@`..`_` (covers SS2, SS3, etc.)
       - Do NOT add a regex crate dependency. Implement as a byte-level scanner.
       - Return the cleaned string.

    Add two new RecipeError variants:
    - `UnknownEffect { name: String }` with Display: "unknown effect: {name}"
    - `PathTraversal { path: String }` with Display: "path traversal rejected: {path}"

    Add `pub mod sandbox;` to `lib.rs`.

    Write tests in `sandbox.rs` `#[cfg(test)] mod tests` covering all behaviors listed above.
  </action>
