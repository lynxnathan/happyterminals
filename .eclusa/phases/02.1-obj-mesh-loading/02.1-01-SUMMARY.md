## Decisions Made

### 1. Flat-normal degeneracy threshold

**Decision:** Use `cross.length() < 1e-12` (absolute, non-squared).

**Rationale:** The research's suggested `length_squared < f32::EPSILON` over-culled real meshes. The Stanford bunny has ~5k triangles with `length_squared(cross) ≈ 1e-8` that are meaningful at its ~0.2-unit scale. Switching to a tight 1e-12 threshold on the unsquared length preserves every real triangle while still rejecting genuine collinear degeneracies (cross = 0 exactly). Discovered during Task 2 TDD when the bunny test returned 0 triangles (see Deviations §1).

### 2. Empty-file contract loosened

**Decision:** `load_obj(empty.obj)` may return `Ok` (with empty mesh) OR `Err`; both satisfy the no-panic contract.

**Rationale:** `tobj 4.0.3` treats 0-byte files as "a file with no models" and returns `Ok(([], _))`. Mapping that to `Err(Parse)` would require extra bookkeeping with no real user benefit — the downstream code already handles empty meshes gracefully. Contract honored as long as no panic occurs.

### 3. flipped_winding fixture is a closed tetrahedron

**Decision:** Instead of a single CW triangle, the fixture is a four-face CW-wound tetrahedron.

**Rationale:** The divergence-theorem signed-volume integral vanishes for any single triangle passing through the origin. A closed (four-face) tetrahedron produces a well-defined negative volume, triggering the warning emission reliably. This matches what real "inside-out" meshes look like in the wild.

### 4. Winding detection emits warnings only (no auto-correction)

**Decision:** `MeshError::Winding` variant is reserved; 2.1 uses `LoadStats.warnings` exclusively.

**Rationale:** Per CONTEXT.md and RESEARCH §Winding Detection — auto-flipping introduces false-positive risk on non-closed meshes (single plane, clipped model), the rasterizer already silhouettes both windings correctly, and users may want inside-out rendering for effect. Keeps the enum variant available for a future opt-in strict mode.

### 5. Mesh.shading kept as a scaffold, never set by loader

**Decision:** `shading: Option<ShadingRamp<'static>>` is always `None` on load.

**Rationale:** Matches D-Per-Model-Shading (CONTEXT.md). The 999.x multi-strategy shading backlog owns the DSL-level `.shading()` / preset selection API; 2.1 only reserves the field so that future work is a non-breaking addition.
