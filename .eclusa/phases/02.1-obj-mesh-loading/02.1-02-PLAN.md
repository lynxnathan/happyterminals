<objective>
Refactor `Renderer::draw` to accept `&Mesh` instead of hardcoded Cube const data, update all in-tree call sites, and extend the `renderer_alloc` criterion bench with a loaded-bunny case that proves zero-per-frame allocations hold at 5000-triangle scale.

Purpose: Plan 01 shipped the `Mesh` type but left the rasterizer pinned to `Cube::VERTICES`/`INDICES`/`FACE_NORMALS`. This plan flips `Renderer::draw` to take `&Mesh` uniformly (REND-06's "unification" requirement from CONTEXT.md) while preserving the zero-allocation discipline (REND-09). After this plan, the backend renders any mesh through the same path — unlocking Plan 03's model-viewer.

Output: Refactored rasterizer, updated backend `render_node` Cube branch, extended bench with `bench_draw_mesh_bunny`, updated existing tests.
</objective>
