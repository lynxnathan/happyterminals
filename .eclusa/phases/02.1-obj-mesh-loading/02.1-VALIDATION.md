# Phase 2.1 — Validation Strategy

> Per-phase validation contract for feedback sampling during execution.

---

## Test Infrastructure

| Property | Value |
|----------|-------|
| **Framework** | Rust `cargo test` + `criterion` (benches) + `proptest` (fuzz) + `insta` (snapshots) — all already configured in `crates/happyterminals-renderer/Cargo.toml` |
| **Config file** | `Cargo.toml` per crate; `insta.yaml` at workspace root |
| **Quick run command** | `cargo test -p happyterminals-renderer mesh` |
| **Full suite command** | `cargo test --workspace && cargo bench -p happyterminals-renderer --bench renderer_alloc -- --test` |
| **Estimated runtime** | ~30 seconds quick; ~90 seconds full suite |

## Sampling Rate

- **After every task commit:** Run `cargo test -p happyterminals-renderer mesh` + `cargo clippy -p happyterminals-renderer -- -D warnings`
- **After every plan wave:** Run `cargo test --workspace && cargo clippy --workspace -- -D warnings`
- **Before `/eclusa:verify-work`:** Full suite must be green including bench-as-test smoke + proptest corpus
- **Max feedback latency:** 30 seconds
