    run() loop:
    - Creates TerminalGuard on entry
    - Creates ratatui Terminal with CrosstermBackend
    - Creates Grid from terminal area
    - tokio::select! between tick interval and EventStream
    - On tick: calls render_fn(&mut grid), copies grid content to frame buffer via terminal.draw()
    - On Ctrl+C: breaks loop, guard drops, terminal restored
    - On resize event: writes new (width, height) to resize signal, grid resizes on next tick
    - On key event: writes InputEvent::Key to last_key_event signal
    - On mouse event: writes InputEvent::Mouse to last_mouse_event signal
    - On focus event: writes bool to focus signal (true=gained, false=lost)

    `pub async fn run<F>(mut render_fn: F, spec: FrameSpec) -> Result<(), Box<dyn std::error::Error>>`
    where `F: FnMut(&mut Grid, &InputSignals)`:

    Note: The render_fn signature takes `&InputSignals` as second argument so scenes can read event signals. This is the BACK-03 wiring.

    1. Call `install_panic_hook()`.
    2. `let _guard = TerminalGuard::acquire()?;`
    3. If `spec.title` is Some, emit `crossterm::execute!(io::stdout(), crossterm::terminal::SetTitle(title))` (best-effort, `let _ = ...`).
    4. Create `CrosstermBackend::new(io::stdout())` + `Terminal::new(backend)?`.
    5. Create `EventStream::new()`.
    6. Create `interval(spec.frame_duration())` with `set_missed_tick_behavior(MissedTickBehavior::Delay)`.
    7. Get initial terminal size: `let (w, h) = crossterm::terminal::size()?;`
    8. Create `let input_signals = InputSignals::new(w, h);`
    9. Create `Grid::new(Rect::new(0, 0, w, h))`.
    10. Loop with `tokio::select!`:
       - Tick branch: `terminal.draw(|frame| { grid.resize(frame.area()); render_fn(&mut grid, &input_signals); *frame.buffer_mut() = grid.deref().clone(); })?;`
         - Add a `tracing::trace_span!("frame")` around the draw call.
       - Event branch: match on mapped events:
         ```rust
         Some(Ok(ev)) => {
             if let Some(input) = map_event(ev) {
                 if is_quit_event(&input) { break; }
                 match &input {
                     InputEvent::Key { .. } => {
                         input_signals.last_key.set(Some(input));
                     }
                     InputEvent::Mouse { .. } => {
                         input_signals.last_mouse.set(Some(input));
                     }
                     InputEvent::Resize { width, height } => {
                         input_signals.terminal_size.set((*width, *height));
                         // Grid resizes on next tick via frame.area()
                     }
                     InputEvent::FocusGained => {
                         input_signals.focused.set(true);
                     }
                     InputEvent::FocusLost => {
                         input_signals.focused.set(false);
                     }
                 }
             }
         }
         ```
    11. Return `Ok(())`. Guard drops here (or on break), restoring terminal.

    **Important note on Grid blit:** The `*frame.buffer_mut() = grid.deref().clone()` copies the entire Grid buffer into ratatui's frame buffer. This requires `Buffer: Clone`. **Executor: verify that `ratatui_core::buffer::Buffer` implements `Clone`.** Check with `cargo doc -p ratatui-core --open` or a quick compile test. If Buffer does NOT implement Clone, fall back to cell-by-cell copy:
    ```rust
    let src = grid.deref();
    let dst = frame.buffer_mut();
    for y in src.area.y..src.area.y+src.area.height {
        for x in src.area.x..src.area.x+src.area.width {
            if let (Some(s), Some(d)) = (src.cell((x,y).into()), dst.cell_mut((x,y).into())) {
                d.set_symbol(s.symbol());
                d.set_style(/* copy style fields */);
                d.skip = s.skip;
            }
        }
    }
    ```
    The clone approach is preferred (simpler, Grid is small: 80x24 = ~96KB at 30fps = trivial).

    **Step 4: 1-cell-change bytes test**

    Create `crates/happyterminals-backend-ratatui/tests/one_cell_change.rs`:

    This is an integration test (not async, no real terminal). Uses `CrosstermBackend::new(Vec::<u8>::new())` as a byte-counting sink.

    ```rust
    use ratatui::Terminal;
    use ratatui_crossterm::CrosstermBackend;
    use ratatui::style::{Color, Style};
    use happyterminals_core::grid::Grid;
    use std::ops::Deref;

    #[test]
    fn one_cell_change_minimal_bytes() {
        // Create terminal with Vec<u8> backend to capture bytes
        let mut buf: Vec<u8> = Vec::new();
        let backend = CrosstermBackend::new(&mut buf);
        let mut terminal = Terminal::new(backend).unwrap();

        // Initial full render
        terminal.draw(|frame| {
            let mut grid = Grid::new(frame.area());
            grid.put_str(0, 0, "Hello, World!", Style::default());
            *frame.buffer_mut() = grid.deref().clone();
        }).unwrap();

        // Clear byte counter after initial render
        let buf_ref = terminal.backend_mut().writer_mut();
        buf_ref.clear();

        // Second render: change exactly one cell
        terminal.draw(|frame| {
            let mut grid = Grid::new(frame.area());
            grid.put_str(0, 0, "Hello, World!", Style::default());
            grid.put_str(0, 0, "X", Style::default().fg(Color::Red));
            *frame.buffer_mut() = grid.deref().clone();
        }).unwrap();

        let delta_bytes = terminal.backend_mut().writer_mut().len();
        // Cursor-move (~6 bytes) + SGR color (~15 bytes) + char (1 byte) + reset
        // CONTEXT.md says ~10 (order of magnitude), RESEARCH.md says actual is ~20-30.
        // Threshold: <= 30 bytes. Full-buffer repaint would be thousands of bytes.
        assert!(
            delta_bytes <= 30,
            "Expected <= 30 bytes for 1-cell change, got {delta_bytes}"
        );
        // Also verify it's not zero (something was actually written)
        assert!(
            delta_bytes > 0,
            "Expected some bytes for 1-cell change, got 0"
        );
    }
    ```

    **Note on Vec<u8> backend:** The test uses `CrosstermBackend::new(&mut buf)` where buf is `Vec<u8>`. The `writer_mut()` method on CrosstermBackend returns `&mut W` (our `&mut Vec<u8>`), giving access to `.len()` and `.clear()`. Verify this API during implementation -- if `writer_mut()` is not available, use an alternative approach (e.g., `Rc<RefCell<Vec<u8>>>` wrapper).

    **Step 5: Event signal propagation test**

    ```rust
    use happyterminals_backend_ratatui::InputSignals;
    use happyterminals_backend_ratatui::InputEvent;
    use happyterminals_core::create_root;
    use crossterm::event::KeyCode;

    #[test]
    fn resize_event_updates_terminal_size_signal() {
        // InputSignals requires reactive runtime context for Signal creation
        create_root(|| {
            let signals = InputSignals::new(80, 24);
            assert_eq!(signals.terminal_size.get(), (80, 24));

            // Simulate resize
            signals.terminal_size.set((120, 40));
            assert_eq!(signals.terminal_size.get(), (120, 40));
        });
    }

    #[test]
    fn key_event_observable_via_signal() {
        create_root(|| {
            let signals = InputSignals::new(80, 24);
            assert!(signals.last_key.get().is_none());

            // Simulate key event written by run()
            signals.last_key.set(Some(InputEvent::Key {
                code: KeyCode::Char('a'),
                modifiers: crossterm::event::KeyModifiers::NONE,
            }));
            let key = signals.last_key.get().unwrap();
            assert!(matches!(key, InputEvent::Key { code: KeyCode::Char('a'), .. }));
        });
    }

    #[test]
    fn focus_event_observable_via_signal() {
        create_root(|| {
            let signals = InputSignals::new(80, 24);
            assert!(signals.focused.get());  // starts focused

            signals.focused.set(false);  // FocusLost
            assert!(!signals.focused.get());

            signals.focused.set(true);   // FocusGained
            assert!(signals.focused.get());
        });
    }
    ```
