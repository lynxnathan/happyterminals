  <behavior>
    run() loop:
    - Creates TerminalGuard on entry
    - Creates ratatui Terminal with CrosstermBackend
    - Creates Grid from terminal area
    - tokio::select! between tick interval and EventStream
    - On tick: calls render_fn(&mut grid), copies grid content to frame buffer via terminal.draw()
    - On Ctrl+C: breaks loop, guard drops, terminal restored
    - On resize event: grid resizes on next tick (frame.area() provides new dims)

    1-cell-change test:
    - Initial full render writes N bytes
    - Changing exactly 1 cell and flushing writes <= 30 bytes
    - This proves ratatui's diff engine works with our Grid (no full repaints)

    **Step 1: run.rs**

    Create `crates/happyterminals-backend-ratatui/src/run.rs`:

    ```rust
    use std::io;
    use tokio::time::{interval, MissedTickBehavior};
    use crossterm::event::EventStream;
    use futures_core::stream::StreamExt;
    use ratatui_crossterm::CrosstermBackend;
    use ratatui::Terminal;

    use happyterminals_core::grid::Grid;
    use crate::guard::{TerminalGuard, install_panic_hook};
    use crate::event::{map_event, is_quit_event};
    use crate::frame_spec::FrameSpec;
    ```

    `pub async fn run<F>(mut render_fn: F, spec: FrameSpec) -> Result<(), Box<dyn std::error::Error>>`
    where `F: FnMut(&mut Grid)`:

    1. Call `install_panic_hook()`.
    2. `let _guard = TerminalGuard::acquire()?;`
    3. If `spec.title` is Some, emit `crossterm::execute!(io::stdout(), crossterm::terminal::SetTitle(title))` (best-effort, `let _ = ...`).
    4. Create `CrosstermBackend::new(io::stdout())` + `Terminal::new(backend)?`.
    5. Create `EventStream::new()`.
    6. Create `interval(spec.frame_duration())` with `set_missed_tick_behavior(MissedTickBehavior::Delay)`.
    7. Create `Grid::new(terminal.get_frame().area())`.
    8. Loop with `tokio::select!`:
       - Tick branch: `terminal.draw(|frame| { grid.resize(frame.area()); render_fn(&mut grid); *frame.buffer_mut() = grid.deref().clone(); })?;`
         - Add a `tracing::trace_span!("frame")` around the draw call.
       - Event branch: `match events.next().await { Some(Ok(ev)) => { if let Some(input) = map_event(ev) { if is_quit_event(&input) { break; } /* resize handled implicitly via frame.area() on next tick */ } }, Some(Err(_)) => break, None => break }`
    9. Return `Ok(())`. Guard drops here (or on break), restoring terminal.

    **Important note on Grid blit:** The `*frame.buffer_mut() = grid.deref().clone()` copies the entire Grid buffer into ratatui's frame buffer. This is intentional -- ratatui then diffs this against the previous frame and emits only changed cells. The Grid is small (typical 80x24 = 1920 cells, ~96KB at 30fps = trivial). The RESEARCH.md confirmed this approach.

    **Step 3: 1-cell-change bytes test**

    Create `crates/happyterminals-backend-ratatui/tests/one_cell_change.rs`:

    This is an integration test (not async, no real terminal). Uses `CrosstermBackend::new(Vec::<u8>::new())` as a byte-counting sink.

    ```rust
    use ratatui::Terminal;
    use ratatui_crossterm::CrosstermBackend;
    use ratatui::style::{Color, Style};
    use happyterminals_core::grid::Grid;
    use std::ops::Deref;

    #[test]
    fn one_cell_change_minimal_bytes() {
        // Create terminal with Vec<u8> backend to capture bytes
        let mut buf: Vec<u8> = Vec::new();
        let backend = CrosstermBackend::new(&mut buf);
        let mut terminal = Terminal::new(backend).unwrap();

        // Initial full render
        terminal.draw(|frame| {
            let mut grid = Grid::new(frame.area());
            grid.put_str(0, 0, "Hello, World!", Style::default());
            *frame.buffer_mut() = grid.deref().clone();
        }).unwrap();

        // Clear byte counter after initial render
        let buf_ref = terminal.backend_mut().writer_mut();
        buf_ref.clear();

        // Second render: change exactly one cell
        terminal.draw(|frame| {
            let mut grid = Grid::new(frame.area());
            grid.put_str(0, 0, "Hello, World!", Style::default());
            grid.put_str(0, 0, "X", Style::default().fg(Color::Red));
            *frame.buffer_mut() = grid.deref().clone();
        }).unwrap();

        let delta_bytes = terminal.backend_mut().writer_mut().len();
        // Order of magnitude: cursor-move (~6 bytes) + SGR color (~15 bytes) + char (1 byte) + reset
        // Should be well under 50 bytes for a single cell change
        assert!(
            delta_bytes <= 50,
            "Expected <= 50 bytes for 1-cell change, got {delta_bytes}"
        );
        // Also verify it's not zero (something was actually written)
        assert!(
            delta_bytes > 0,
            "Expected some bytes for 1-cell change, got 0"
        );
    }
    ```

    **Note on byte threshold:** CONTEXT.md says "~10 bytes" but RESEARCH.md clarifies actual ANSI output for cursor-move + SGR + char is ~20-30 bytes. Use 50 as a generous upper bound that still proves no full-buffer repaint (which would be thousands of bytes). The exact number will be documented in the SUMMARY.

    **Note on Vec<u8> backend:** The test uses `CrosstermBackend::new(&mut buf)` where buf is `Vec<u8>`. The `writer_mut()` method on CrosstermBackend returns `&mut W` (our `&mut Vec<u8>`), giving access to `.len()` and `.clear()`. Verify this API during implementation -- if `writer_mut()` is not available, use an alternative approach (e.g., `Rc<RefCell<Vec<u8>>>` wrapper).
