## Decisions Made
- **1-cell-change threshold:** Adjusted from plan's 30 bytes to 50 bytes. Empirically measured at 44 bytes (cursor move + SGR + char + reset). 50 bytes still proves differential rendering (full repaint would be thousands).
- **current_thread runtime:** Static demo uses `tokio::main(flavor = "current_thread")` because workspace tokio features don't include `rt-multi-thread`. Single-threaded is correct for terminal apps anyway (Signal is !Send).
- **SharedBuf test pattern:** Used `Rc<RefCell<Vec<u8>>>` implementing `Write` for byte counting, since `CrosstermBackend::writer_mut()` is behind an unstable feature gate in ratatui-crossterm 0.1.0.
