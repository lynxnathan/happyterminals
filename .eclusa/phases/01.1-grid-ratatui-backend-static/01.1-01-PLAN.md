    **Step 3: cell.rs**

    Create `crates/happyterminals-core/src/cell.rs`. Our Cell is a thin semantic wrapper providing display_width info and grapheme-cluster documentation. Since Grid::put_str writes directly into ratatui Cells via `Buffer::cell_mut()`, our Cell type serves as the public API type for reading cell state:

    ```rust
    use ratatui_core::buffer::Cell as RatatuiCell;
    use ratatui_core::style::{Color, Modifier, Style};

    /// A single terminal cell. Wraps ratatui's Cell with explicit
    /// display-width semantics for grapheme clusters.
    ///
    /// Cells are not constructed directly -- they are written via
    /// [`Grid::put_str`] and read via `Grid::deref() -> &Buffer`.
    pub struct Cell;  // Zero-sized marker / namespace for associated fns
    ```

    Cell provides associated functions:
    - `pub fn display_width(s: &str) -> usize` -- uses `unicode_width::UnicodeWidthStr::width(s)` to return the display width of a grapheme cluster string.
    - `pub fn graphemes(s: &str) -> impl Iterator<Item = &str>` -- delegates to `unicode_segmentation::UnicodeSegmentation::graphemes(s, true)`.

    Since ratatui's Buffer stores `ratatui_core::buffer::Cell` directly and we access them via `Deref<Target=Buffer>`, we do NOT wrap ratatui::Cell in our own struct. Instead, Cell is a utility namespace. Re-export `ratatui_core::buffer::Cell as RatatuiCell` for direct cell reads, and re-export `ratatui_core::style::{Color, Modifier, Style}` for users.

    **Step 4: grid.rs**

    Create `crates/happyterminals-core/src/grid.rs`:

    ```rust
    use ratatui_core::buffer::Buffer;
    use ratatui_core::layout::Rect;
    use ratatui_core::style::Style;
    use unicode_segmentation::UnicodeSegmentation;
    use unicode_width::UnicodeWidthStr;

    pub struct Grid {
        inner: Buffer,
    }

    impl std::ops::Deref for Grid {
        type Target = Buffer;
        fn deref(&self) -> &Buffer { &self.inner }
    }

    // NO DerefMut -- writes go through put_str only
    ```

    Methods:
    - `pub fn new(area: Rect) -> Self` -- `Self { inner: Buffer::empty(area) }`
    - `pub fn put_str(&mut self, x: u16, y: u16, s: &str, style: Style)` -- iterates grapheme clusters via `s.graphemes(true)`, computes display width via `grapheme.width()`, writes primary cell via `self.inner.cell_mut(Position::new(col, y))?.set_symbol(grapheme).set_style(style)`, marks continuation cells with `reset() + skip = true`. Silently clips out-of-bounds (checks y in area range, col in area range before each write). Zero-width graphemes (combining marks) are skipped (they attach to previous character via the grapheme iterator).
    - `pub fn resize(&mut self, area: Rect)` -- `self.inner = Buffer::empty(area);`
    - `pub(crate) fn inner_mut(&mut self) -> &mut Buffer` -- for backend blit access
