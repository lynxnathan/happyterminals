# Phase 01.1: Grid + Ratatui Backend (static) - Research

**Researched:** 2026-04-15
**Domain:** Grid buffer model (grapheme-cluster-correct) + ratatui/crossterm backend event loop with RAII terminal guard
**Confidence:** HIGH

## Summary

This phase builds two foundational pieces: (1) a `Cell` + `Grid` type in `happyterminals-core` that correctly handles grapheme clusters and wide characters, newtypes over `ratatui::Buffer`, and (2) a panic-safe `TerminalGuard` + `run()` event loop in `happyterminals-backend-ratatui` using tokio + crossterm's `EventStream`.

The ratatui 0.30 `Buffer` struct has **public** `area: Rect` and `content: Vec<Cell>` fields, making a newtype via `Deref`/`DerefMut` straightforward. The ratatui `Cell` stores its symbol as a private `CompactString` field (accessible via `symbol()` / `set_symbol(&str)`) plus public `fg`, `bg`, `underline_color`, `modifier`, and `skip` fields. Our `Grid` wraps `Buffer` and adds `put_str(x, y, s, style)` which segments input via `unicode-segmentation` and measures width via `unicode-width`. Wide characters occupy a primary cell plus a `skip=true` continuation cell -- the same model ratatui uses internally.

For the 1-cell-change bytes test (GRID-04), `TestBackend` does NOT expose raw bytes; instead use `CrosstermBackend::new(Vec::<u8>::new())` to capture actual ANSI output and measure byte count after a single-cell mutation via `terminal.flush()`.

**Primary recommendation:** Newtype `Grid` over `ratatui::Buffer` with `Deref<Target=Buffer>` for read access and controlled mutation through `put_str`. Start with a 1-2 day spike to validate layout compat. Use tokio 1.52 for the async runtime, crossterm `event-stream` feature for `EventStream`.

### Pattern 1: Grid as Deref Newtype over ratatui::Buffer

**What:** Grid wraps Buffer, exposes Buffer's read API via Deref, controls writes through put_str only.

**When to use:** Always -- Grid is the sole way to write content.

**Example:**
```rust
// Source: ratatui docs + project CONTEXT.md decisions

use ratatui::buffer::Buffer;
use ratatui::layout::Rect;
use ratatui::style::Style;
use unicode_segmentation::UnicodeSegmentation;
use unicode_width::UnicodeWidthStr;

pub struct Grid {
    inner: Buffer,
}

impl std::ops::Deref for Grid {
    type Target = Buffer;
    fn deref(&self) -> &Buffer { &self.inner }
}

impl Grid {
    pub fn new(area: Rect) -> Self {
        Self { inner: Buffer::empty(area) }
    }

    /// Sole text-writing API. Segments `s` into grapheme clusters,
    /// computes display width, and writes cells. Wide chars occupy
    /// a primary cell + skip continuation cell. Out-of-bounds silently clips.
    pub fn put_str(&mut self, x: u16, y: u16, s: &str, style: Style) {
        let mut col = x;
        let area = self.inner.area;
        for grapheme in s.graphemes(true) {
            let width = grapheme.width() as u16;
            if width == 0 { continue; } // combining marks attach to previous
            if y < area.y || y >= area.y + area.height { return; }
            if col >= area.x + area.width { return; } // clip right edge

            // Set primary cell
            if let Some(cell) = self.inner.cell_mut((col, y).into()) {
                cell.set_symbol(grapheme);
                cell.set_style(style);
            }

            // Wide char: mark continuation cell(s) as skip
            for offset in 1..width {
                if col + offset < area.x + area.width {
                    if let Some(cell) = self.inner.cell_mut((col + offset, y).into()) {
                        cell.reset();
                        cell.skip = true;
                    }
                }
            }

            col += width;
        }
    }

    /// Resize grid for new terminal dimensions.
    pub fn resize(&mut self, area: Rect) {
        self.inner = Buffer::empty(area);
    }

    /// Mutable access to inner buffer for backend blit.
    pub(crate) fn inner_mut(&mut self) -> &mut Buffer {
        &mut self.inner
    }
}
```

### Pattern 2: TerminalGuard RAII + Panic Hook

**What:** Guard struct that enables raw mode / alt screen / mouse / hides cursor on creation, restores on Drop.

**When to use:** Always created at the start of run(), dropped at end or on panic.

**Example:**
```rust
// Source: PITFALLS.md #1, #31 + crossterm docs

use std::io::{self, Write, Stdout};
use crossterm::{
    execute,
    terminal::{self, EnterAlternateScreen, LeaveAlternateScreen},
    cursor,
    event::{EnableMouseCapture, DisableMouseCapture},
};

pub struct TerminalGuard {
    stdout: Stdout,
}

impl TerminalGuard {
    pub fn acquire() -> io::Result<Self> {
        let mut stdout = io::stdout();
        terminal::enable_raw_mode()?;
        execute!(
            stdout,
            EnterAlternateScreen,
            EnableMouseCapture,
            cursor::Hide,
        )?;
        Ok(Self { stdout })
    }

    fn restore(stdout: &mut Stdout) {
        // Best-effort: ignore errors during cleanup
        let _ = execute!(
            stdout,
            cursor::Show,
            DisableMouseCapture,
            LeaveAlternateScreen,
        );
        let _ = terminal::disable_raw_mode();
        // Reset SGR to prevent color leakage into shell
        let _ = stdout.write_all(b"\x1b[0m");
        let _ = stdout.flush();
    }
}

impl Drop for TerminalGuard {
    fn drop(&mut self) {
        Self::restore(&mut self.stdout);
    }
}

// Panic hook: install before creating guard
pub fn install_panic_hook() {
    let original = std::panic::take_hook();
    std::panic::set_hook(Box::new(move |info| {
        // Restore terminal before printing panic
        let mut stdout = io::stdout();
        TerminalGuard::restore(&mut stdout);
        original(info);
    }));
}
```

### Pattern 3: tokio::select! Event Loop

**What:** Frame ticker + input event stream in a select! loop.

**Example:**
```rust
// Source: ARCHITECTURE.md #10.1 + crossterm EventStream docs

use tokio::time::{interval, MissedTickBehavior};
use crossterm::event::{EventStream, Event};
use futures_core::stream::StreamExt;

pub struct FrameSpec {
    pub fps: u32,
    pub title: Option<String>,
    pub mouse_capture: bool,
}

impl Default for FrameSpec {
    fn default() -> Self {
        Self { fps: 30, title: None, mouse_capture: true }
    }
}

pub async fn run<F>(mut render_fn: F, spec: FrameSpec) -> Result<(), Box<dyn std::error::Error>>
where
    F: FnMut(&mut Grid),
{
    install_panic_hook();
    let _guard = TerminalGuard::acquire()?;

    let backend = ratatui_crossterm::CrosstermBackend::new(io::stdout());
    let mut terminal = ratatui::Terminal::new(backend)?;
    let mut events = EventStream::new();

    let frame_dur = std::time::Duration::from_millis(1000 / spec.fps as u64);
    let mut tick = interval(frame_dur);
    tick.set_missed_tick_behavior(MissedTickBehavior::Delay);

    let mut grid = Grid::new(terminal.get_frame().area());

    loop {
        tokio::select! {
            _ = tick.tick() => {
                // Render frame
                terminal.draw(|frame| {
                    grid.resize(frame.area());
                    render_fn(&mut grid);
                    // Blit grid into frame buffer
                    let buf = frame.buffer_mut();
                    // Copy grid content into ratatui's buffer
                    *buf = grid.inner.clone();
                })?;
            }
            ev = events.next() => {
                match ev {
                    Some(Ok(Event::Key(key))) => {
                        use crossterm::event::KeyCode;
                        if key.code == KeyCode::Char('c')
                            && key.modifiers.contains(
                                crossterm::event::KeyModifiers::CONTROL)
                        {
                            break;
                        }
                    }
                    Some(Ok(Event::Resize(w, h))) => {
                        // Grid will resize on next tick via frame.area()
                    }
                    Some(Ok(_)) => {}
                    Some(Err(_)) => break,
                    None => break,
                }
            }
        }
    }

    Ok(())
}
```

### Anti-Patterns to Avoid

- **Grid API takes `char` instead of `&str`:** Breaks grapheme cluster support. Cell must hold a string slice, not a single char.
- **Double-buffering inside Grid:** ratatui's Terminal already double-buffers. Adding another layer wastes memory and complicates the diff.
- **Calling `terminal.clear()` every frame:** Defeats ratatui's diff engine. Never clear -- just write to the buffer and let ratatui diff.
- **Reading `terminal::size()` multiple times per frame:** Resize race. Read once between frames.
- **Deref + DerefMut both exposed:** DerefMut would let callers bypass put_str and write directly to Buffer cells, losing grapheme safety. Only expose Deref for read access.

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| ANSI diff engine | Custom cell-diff-to-ANSI writer | ratatui Terminal::flush() | ratatui's diff is battle-tested, handles cursor movement, SGR state tracking, skip cells |
| Grapheme segmentation | Manual codepoint-level parsing | unicode-segmentation crate | UAX#29 is complex; ZWJ sequences, regional indicators, combining marks |
| Display width calculation | `str::len()` or `chars().count()` | unicode-width crate | Width != bytes != codepoints; CJK = 2, combining = 0 |
| Terminal state restoration | Manual cleanup calls | TerminalGuard Drop impl + panic hook | Drop guarantees cleanup on all exit paths including panic |
| Frame timing | Manual Instant arithmetic | tokio::time::interval with MissedTickBehavior::Delay | Handles clock drift, missed ticks, sleep precision |
| Async event stream | Polling crossterm::event::read() in a thread | crossterm EventStream (event-stream feature) | Native Stream impl, cancellation-safe in tokio::select! |

**Key insight:** ratatui already solved the hard terminal rendering problems (diff engine, double buffering, cell model). Grid is a thin semantic layer on top, not a replacement.

## Common Pitfalls

### Pitfall 1: Terminal Left in Raw Mode After Panic
**What goes wrong:** Panic anywhere in render loop leaves terminal unusable -- no echo, cursor invisible, alt screen active.
**Why it happens:** crossterm's enable_raw_mode / EnterAlternateScreen are not RAII by default.
**How to avoid:** TerminalGuard with Drop impl + panic hook installed before any terminal operations.
**Warning signs:** Bare `enable_raw_mode()` call without matching guard.

### Pitfall 2: Unicode Width Misalignment
**What goes wrong:** Emoji/CJK after a wide char are offset by 1 column; 3D scenes warp.
**Why it happens:** Using `str::len()`, `chars().count()`, or codepoint-level width instead of grapheme-cluster + display-width.
**How to avoid:** All text goes through `Grid::put_str` which uses `unicode-segmentation` for grapheme clusters and `unicode-width` for display width. No other code path writes text.
**Warning signs:** Grid API takes `char` instead of `&str`; demos use ASCII only.

### Pitfall 3: Resize Race Condition
**What goes wrong:** Rendering against stale dimensions after resize -- scene wraps or draws into void.
**Why it happens:** SIGWINCH is async; resize can fire mid-frame.
**How to avoid:** Resize events drain between frames in the select! loop; Grid resizes at start of next frame tick; never during rasterization. Re-query `crossterm::terminal::size()` after resize event.
**Warning signs:** Reading terminal size more than once per frame; no explicit resize handling.

### Pitfall 4: Full-Buffer Redraw Defeating Reactivity
**What goes wrong:** Despite fine-grained signals, calling `terminal.clear()` or re-rendering entire buffer every frame.
**Why it happens:** Easier than tracking which cells changed.
**How to avoid:** Never call terminal.clear(). Let ratatui's double-buffer diff determine what changed. The 1-cell-change bytes test (GRID-04) enforces this.
**Warning signs:** Per-frame byte count is constant regardless of what changed.

### Pitfall 5: SGR State Leakage After Exit
**What goes wrong:** After exiting, shell prompt shows wrong colors or mouse events still captured.
**Why it happens:** Terminal is a global state machine; SGR sequences persist until reset.
**How to avoid:** TerminalGuard cleanup must include `\x1b[0m` (reset SGR), show cursor, disable all mouse modes, leave alt screen, disable raw mode.
**Warning signs:** After running demo, next shell prompt is colored; mouse clicks emit sequences.

### Pitfall 6: crossterm Missing event-stream Feature
**What goes wrong:** `crossterm::event::EventStream` not found at compile time.
**Why it happens:** EventStream requires `features = ["event-stream"]` on the crossterm dependency. Current workspace dep lacks this feature.
**How to avoid:** Update workspace `crossterm` dep to include `features = ["event-stream"]`.
**Warning signs:** Compile error on `use crossterm::event::EventStream`.

## Code Examples

### 1-Cell-Change Bytes Test (GRID-04)

```rust
// Source: ratatui CrosstermBackend docs + CONTEXT.md decision

use ratatui::Terminal;
use ratatui_crossterm::CrosstermBackend;
use ratatui::style::{Color, Style};

#[test]
fn one_cell_change_minimal_bytes() {
    let mut buf = Vec::<u8>::new();
    let backend = CrosstermBackend::new(&mut buf);
    let mut terminal = Terminal::new(backend).unwrap();

    // Initial full render
    terminal.draw(|frame| {
        let area = frame.area();
        let mut grid = Grid::new(area);
        grid.put_str(0, 0, "Hello, World!", Style::default());
        *frame.buffer_mut() = grid.inner.clone();
    }).unwrap();

    let initial_bytes = buf.len();
    buf.clear();

    // Mutate exactly one cell
    terminal.draw(|frame| {
        let area = frame.area();
        let mut grid = Grid::new(area);
        grid.put_str(0, 0, "Hello, World!", Style::default());
        // Change one cell: 'H' -> 'X'
        grid.put_str(0, 0, "X", Style::default().fg(Color::Red));
        *frame.buffer_mut() = grid.inner.clone();
    }).unwrap();

    let delta_bytes = buf.len();
    // Should be small: ~cursor-move + SGR + char
    // Order of magnitude: <= ~30 bytes (generous; exact depends on cursor position encoding)
    assert!(
        delta_bytes <= 30,
        "Expected <= 30 bytes for 1-cell change, got {delta_bytes}"
    );
}
```

**Note:** The exact byte count depends on cursor position encoding and SGR state. The CONTEXT.md says "~10" as order of magnitude. In practice, a cursor-move (CSI H = ~6 bytes) + SGR color (CSI 38;2;R;G;Bm = ~15 bytes) + char (1 byte) + SGR reset = roughly 20-30 bytes. The test should use an order-of-magnitude bound, not an exact match. Validate the exact number during the spike.

## Open Questions

1. **Grid newtype Deref vs. blit-copy**
   - What we know: Buffer fields are public; Deref newtype is clean. However, ratatui's `Terminal::draw` provides a `Frame` whose `buffer_mut()` returns `&mut Buffer`. We need to get our Grid content into that buffer.
   - What's unclear: Whether to clone Grid.inner into Frame's buffer (simple but copies) or use a different draw approach (call `set_string` on the frame buffer directly from Grid).
   - Recommendation: The spike should test both approaches. Clone is simplest and the Grid is small (80x24 = 1920 cells * ~50 bytes = ~96KB). At 30fps this is trivial.

2. **Exact 1-cell-change byte threshold**
   - What we know: CONTEXT.md says "~10 bytes." Actual ANSI output for cursor-move + SGR + char is more like 20-30 bytes.
   - What's unclear: Whether ratatui optimizes cursor movement when position hasn't changed much.
   - Recommendation: Spike determines exact number; adjust test threshold to match reality. The point is order-of-magnitude, not exact match.

3. **FrameSpec: title field behavior**
   - What we know: We want an optional window title.
   - What's unclear: Whether to use crossterm's `SetTitle` command and if all terminals support it.
   - Recommendation: Include in FrameSpec, apply via crossterm `SetTitle` if Some, document as best-effort.
