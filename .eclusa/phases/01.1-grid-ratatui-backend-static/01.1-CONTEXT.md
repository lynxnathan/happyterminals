## Phase Boundary

**Goal:** Ship a grapheme-cluster-correct `Grid` (newtyped over `ratatui::Buffer`) and a panic-safe `-backend-ratatui` event loop that can render a static Grid and exit cleanly on Ctrl-C or panic. No animation yet — "pixels on screen, safely."

**In scope:**
- `happyterminals-core`: add `Cell` + `Grid` types (grapheme-cluster-correct)
- `happyterminals-backend-ratatui`: `TerminalGuard` RAII + panic hook, `run()` with tokio::select loop, input→signal propagation, resize handling
- `happyterminals` meta crate: curated re-exports / prelude
- Grid/Buffer layout-compat spike (1-2 days, gates the newtype approach)
- 1-cell-change bytes test (M1 exit gate foundation)
- Static demo binary: render grapheme text + emoji + CJK, exit on Ctrl-C

**Out of scope:**
- Animation / frame tick driving scene state — Phase 1.2+
- Pipeline / effects — Phase 1.2
- 3D rendering — Phase 1.3
- Python bindings — Phase 4.x
- Color fallback pipeline (RGB→256→16→mono) — Phase 2.2
- WASM — explicitly not M1

<decisions>
## Implementation Decisions (locked)

### Grid design (GRID-01, GRID-02)
- **Grid is a newtype over `ratatui::Buffer`** (ARCH §5.2, §6.3). A 1-2 day spike verifies layout compat before committing. Fallback: own-struct Grid with `impl From<&Grid> for ratatui::Buffer`.
- `Cell` holds one grapheme cluster (not `char`), display width, fg/bg colors (ratatui's `Color` enum covers RGB/256/16), and modifiers (Bold/Italic/Underline/Reverse via ratatui's `Modifier`).
- `Grid::put_str(x, y, s, style)` is the sole text-writing API. Handles multi-byte, wide (east-asian, emoji), combining marks, ZWJ sequences via `unicode-segmentation` + `unicode-width`.

### Ratatui backend (BACK-01..04)
- `TerminalGuard` RAII struct: on creation enters raw mode + alternate screen + enables mouse capture + hides cursor. On drop (including panic) restores all. Panic hook installed to ensure drop runs.
- `run(scene_fn, FrameSpec)` drives a `tokio::select!` loop:
  - `tick.tick()` fires at `FrameSpec.fps` (default 30)
  - `crossterm::EventStream::next()` for input
  - All frame work synchronous on the render thread
- Input events → signals: key, mouse, resize, focus events map to signals the scene can read.
- Resize: events drained between frames; Grid dimensions update on next frame; no resize mid-rasterization.

### Async runtime (Q2 — resolve NOW)
- **Decision: tokio** (ecosystem weight, PyO3 story via `pyo3-async-runtimes` which defaults to tokio, crossterm's `EventStream` is tokio-based).
- Add `tokio` to `[workspace.dependencies]` with `features = ["rt", "time", "macros"]`.
- Only `-backend-ratatui` depends on tokio; core crate stays runtime-agnostic.

### Dependencies to activate
In `crates/happyterminals-backend-ratatui/Cargo.toml`:
- `happyterminals-core = { path = "../happyterminals-core" }`
- `ratatui = { workspace = true }` (full facade, crossterm feature)
- `ratatui-crossterm = { workspace = true }`
- `crossterm = { workspace = true }`
- `tokio = { workspace = true }`

In `crates/happyterminals-core/Cargo.toml` (additions):
- `ratatui-core = { workspace = true }` (for Buffer/Cell interop — lib-level dep, not the full facade)
- `unicode-segmentation = "1.11"` or `unicode-width = "0.2"` (add to workspace deps)

In root `Cargo.toml` `[workspace.dependencies]` (additions):
- `tokio = { version = "1", features = ["rt", "time", "macros"] }`
- `unicode-segmentation = "1.11"`
- `unicode-width = "0.2"`

### 1-cell-change bytes test (GRID-04)
- After rendering a full Grid to a `ratatui::Terminal<TestBackend>`, mutate exactly one cell via `Grid::put_str`, flush again, and assert the total bytes written is ≤ ~10 (order of magnitude — ratatui's diff produces a single cursor-move + SGR + char for one cell).
- This is the M1 exit gate foundation — Phase 1.5 will re-run this as a hard gate.

### Meta crate (BACK-04)
- `crates/happyterminals/src/lib.rs` re-exports: `prelude` module with `Signal`, `Memo`, `Effect`, `Owner`, `create_root`, `on_cleanup`, `batch`, `Grid`, `Cell`, `run`, `FrameSpec`.
- Users write `use happyterminals::prelude::*;`.

### Claude's Discretion
- Exact `ratatui::Buffer` newtype approach (direct field access via `Deref` vs conversion methods).
- Whether to include `tracing` spans at the frame-tick boundary (recommend: yes, one span per frame).
- Exact `FrameSpec` struct fields (fps, title, mouse_capture).
- Whether `Grid::put_str` returns a `Result` or panics on out-of-bounds (recommend: silently clips, like ratatui).

</decisions>
