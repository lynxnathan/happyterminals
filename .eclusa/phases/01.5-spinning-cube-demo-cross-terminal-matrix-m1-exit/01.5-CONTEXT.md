## Implementation Decisions (locked)

### Scene → run() wiring (DEMO-01)
- **D-01:** Add `pub async fn run_scene(scene: Scene, spec: FrameSpec) -> Result<...>` to the backend crate. This function: creates the reactive root, extracts camera from scene, creates Renderer, and runs the frame loop. Each tick: walks SceneIr nodes, calls Renderer::draw() per Cube node with node's transform, applies Pipeline effects, copies Grid to ratatui frame.
- **D-02:** [auto] The existing `run()` (closure-based) stays for backward compat. `run_scene()` is the new Scene-aware entry point. Both are re-exported from the prelude.
- **D-03:** Signal-driven rotation: the spinning cube's azimuth signal is incremented each frame tick (e.g., `azimuth.set(azimuth.untracked() + dt * rotation_speed)`). The OrbitCamera reads it via `.untracked()` in the render callback.

### Example (DEMO-01, DEMO-02)
- **D-04:** `examples/spinning-cube/main.rs` uses the DSL builder:
  ```rust
  let scene = scene()
      .camera(orbit_camera)
      .layer(0, |l| l.cube().rotation(&azimuth_signal))
      .pipeline(Pipeline::new().with_boxed(effects::dissolve(Duration::from_secs(0))))
      .build()?;
  run_scene(scene, FrameSpec::default()).await
  ```
- **D-05:** < 100 LOC including imports, comments, and main function. Uses `happyterminals::prelude::*` for minimal imports.

## Existing Code Insights

### What needs wiring
- `run()` currently takes `FnMut(&mut Grid, &InputSignals)` — need `run_scene(Scene, FrameSpec)`
- Scene has nodes but nothing walks them to call Renderer::draw()
- OrbitCamera exists in renderer but Scene owns CameraConfig::Orbit(OrbitCamera) — bridge needed
- Pipeline exists but isn't called in the render loop yet
- All 164 tests pass — integration must not break them

### Integration path
1. `run_scene()` creates reactive root, acquires TerminalGuard
2. Extracts OrbitCamera from Scene.camera()
3. Creates Renderer with Projection
4. Frame loop: for each SceneNode of kind Cube, call Renderer::draw() with node transform
5. After all nodes rendered, apply Scene's Pipeline to the Grid
6. Copy Grid to ratatui frame (existing pattern)
7. Input events → signals (existing pattern)
8. Signal updates drive rotation (azimuth increment per frame)
