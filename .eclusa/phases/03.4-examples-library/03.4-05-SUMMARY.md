## Decisions Made

- **Compile-only smoke (not runtime):** CI runners (GitHub Actions ubuntu-latest) are TTY-less, and crossterm's `enable_raw_mode()` errors on non-TTY file descriptors (RESEARCH Pitfall 6). `cargo run --example` would crash before user code runs. `cargo check --example` catches type drift, missing asset paths, feature gating, and prelude gaps — the failure modes most likely to break the 5 runnable-in-a-fresh-checkout promise. Runtime smoke is deferred to a post-phase follow-up that either adds `HT_SMOKE_EXIT_AFTER_MS` or wraps with a PTY emulator (per RESEARCH F-D-13 and §Finding 4).
- **Fail-fast over aggregated reporting:** `set -euo pipefail` on both scripts. The first failure is always the interesting one and aggregated mode adds shell complexity with no plan-verification upside (RESEARCH open Q #3 recommendation).
- **Dev-utility allowlist hardcoded:** `DEV_UTILITIES=("color-test" "static_grid")` is a 2-entry array rather than a file-content probe. Simpler, obvious, and the list is stable — any new dev utility in the future can be added with a 1-line change.
- **spinning-cube treated as headline by the linter:** It's NOT a headline per D-02 but it DOES carry the full DEMO-05 header shape (verified during Plan 04). The doc-lint treats it as headline because it's not on the dev-utility allowlist — which is fine, since the header contract passes. No special case needed.
- **YAML appended, not inserted:** The new job lives at the end of `jobs:`, after `hygiene`. Zero risk of accidentally modifying another job. YAML still parses cleanly (`python3 -c 'import yaml; yaml.safe_load(...)'` exits 0) and job count is 10 (6 visible top-level job keys including `examples-smoke`, plus nested keys under `jobs:` — the criterion `$1 >= 6` is satisfied with room to spare).
