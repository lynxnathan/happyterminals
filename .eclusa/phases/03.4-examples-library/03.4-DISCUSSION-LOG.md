| Option | Description | Selected |
|--------|-------------|----------|
| Simple load+render | One hello.json → scene; minimal surface | |
| Happy path + sandbox demo | Load hello.json via `load_recipe_sandboxed`; include a commented rejected-path example that prints to stderr | ✓ |

**Selection rationale (auto-selected recommended):** Demonstrates both the round-trip AND the security story shipped in Phase 3.3, without overbuilding into an editor. The commented-out bad path educates readers about the sandbox without requiring them to craft an attack to see it fire.

## Runnability Gate

| Option | Description | Selected |
|--------|-------------|----------|
| `cargo check --examples` only | Ensures they compile | |
| Runtime smoke test per example (short timeout) | Starts each, kills after N seconds, passes if no panic | ✓ |
| Full interactive test matrix | Automated UI assertions | |

**Selection rationale (auto-selected recommended):** A compile gate misses runtime panics (mis-typed asset paths, missing env setup). Runtime smoke gives real confidence without the cost of full UI assertions. Runs in CI.
