The two newly-authored examples (text-reveal and json-loader) need human TTY verification because the plan explicitly replaced runtime smoke with compile-only smoke (RESEARCH F-D-13 + Pitfall 6: crossterm raw-mode fails on TTY-less CI runners). #### 1. HAT: text-reveal renders bunny + animated title overlay

**Run:**

```bash
cargo run --example text-reveal -p happyterminals
```

**Expected:**
- A rotating bunny mesh renders continuously below the centered title region.
- On launch, three lines appear inside the title rect (centered horizontally, row ~4):
  - "HAPPY TERMINALS" (bold white)
  - "GPU-quality effects on text" (cyan)
  - ">  press [space] to continue" (yellow)
- The text animates in via a bounded tachyonfx effect (default: `fade_from`).
- The text effect does NOT bleed onto the bunny's shading (bounded to title rect).
- Pressing **Space** replays the reveal from t=0.
- Pressing **Tab** cycles to the next effect: `fade_from` → `sweep_in` → `coalesce` → back.
- **Q** or **Ctrl-C** quits cleanly (no terminal artefacts; cursor restored).

**Why human:** Visual composition (a GPU-quality effect driving text over a 3D mesh rendered in ASCII) IS the hero paradigm this example exists to demonstrate. Per project memory `feedback_visual_iteration`, snapshot tests don't substitute for human judgment on hero visuals.

#### 2. HAT: json-loader renders the cube described by hello.json

**Run:**

```bash
cargo run --example json-loader -p happyterminals
```

**Expected:**
- A single cube renders with the camera pose from `examples/recipes/hello.json`:
  - azimuth 0.5, elevation 0.3, distance 5.0, target (0, 0, 0)
- No sandbox rejection appears on stderr (the path-traversal demo is commented out by default per D-07).
- Static scene — no per-frame animation.
- **Q** or **Ctrl-C** quits cleanly.
- *(Optional verification step)* Uncomment the rejected-path block in `main.rs` (lines 69–82) and re-run; expect a single stderr line matching `sandbox blocked path traversal: ../../etc/passwd` and the cube still renders.

**Why human:** Terminal rendering of a JSON-described scene needs a real TTY. CI blocks runtime smoke on TTY-less runners per RESEARCH F-D-13 / Pitfall 6; per the plan's VALIDATION.md the equivalent gate is human visual confirmation.
