## Decisions Made
- **Mirror the DSL prelude grouping.** The new `pub use happyterminals_dsl::json::{...}` block lists the two new names inline with the four existing ones (alphabetical within the group), and `EffectRegistry` is re-exported on its own line immediately below because it lives in a different submodule (`::sandbox`). This matches `crates/happyterminals-dsl/src/prelude.rs` exactly, keeping the two preludes visually aligned so future additions can be copied across with minimal thinking.
- **Fix only the clippy issues in the edited files + the DSL errors that directly block clippy verification.** The 5 remaining clippy errors surfaced in `examples/model-viewer/main.rs` after unblocking the DSL compile; they are pre-existing (Phase 2.3 / 2.5 territory) and out of scope per the SCOPE BOUNDARY rule. Documented for Phase 03.5 pre-publish cleanup.
- **Restructure the test rather than adding `#[allow(clippy::items_after_statements)]`.** Moving `fn _check_types` above the two `let` bindings is a cleaner refactor than annotating, and since the plan already asked to modify this exact test, the restructure lives with the additions in one atomic commit.

## Issues Encountered
- **`git stash` side-effect during verification.** While investigating whether the clippy errors were pre-existing, a `git stash` operation was used to check the pre-change state. Because Cargo.lock had uncommitted orchestrator changes, the stash pop initially conflicted. Resolved by resetting Cargo.lock to HEAD and popping cleanly. No code was lost — the diff was verified afterwards. Lesson: when using `git stash` inside an executor, scope the stash to the files being investigated (`git stash push <paths>`) rather than a whole-tree stash.
