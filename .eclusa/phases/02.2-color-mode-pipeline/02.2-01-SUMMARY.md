## Decisions Made

See frontmatter `key-decisions`. Most consequential:

1. **`EnvProvider` trait, not `temp_env` / `serial_test`** — closes Pitfall 4 with zero new deps and better unit-test purity (cascade becomes a pure function of env). Future phases that read env can reuse this pattern.
2. **`ratatui::` re-exports over `ratatui-core::` paths** — discovered during Task 2 build (Rule 3 blocker). Backend-ratatui does not name `ratatui-core` directly in its `[dependencies]`; it must consume types via the `ratatui` re-export surface. Consistent with pre-existing backend code in `run.rs`, `event.rs`, `tests/e2e_phase_1_1.rs`.
3. **Mono → `Color::Reset` (Pitfall 7 gate)** — explicit inline test + comment; prevents the seductive-but-wrong `Color::Black` mapping that breaks on light-theme terminals.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] `ratatui_core` not in backend-ratatui `[dependencies]`**
- **Found during:** Task 2 (first build after writing `use ratatui_core::...` per plan `<interfaces>` block).
- **Issue:** Plan's `<interfaces>` used `ratatui_core::buffer::Buffer` paths, but backend-ratatui consumes these types via the re-exported `ratatui::buffer::Buffer` surface. Pre-existing test/src code uses `ratatui::` paths.
- **Fix:** Changed `use ratatui_core::{buffer::Buffer, layout::Position, style::Color};` to `use ratatui::{buffer::Buffer, layout::Position, style::Color};` — `Buffer`/`Position`/`Color` are re-exports of the same types.
- **Files modified:** `src/color.rs`.
- **Verification:** `cargo build -p happyterminals-backend-ratatui` clean; tests compile + pass.
- **Committed in:** `43f9dfa` (Task 2 commit).

**2. [Rule 1 - Bug] `buffer.area()` borrow-checker conflict**
- **Found during:** Task 2 (first build after writing the downsample loop).
- **Issue:** `let area = buffer.area();` holds an immutable borrow on `buffer` through the nested loop, preventing `buffer.cell_mut(...)` calls. `buffer.area()` in ratatui 0.30 returns `&Rect` not `Rect`.
- **Fix:** Copy the `Rect` out with `let area = *buffer.area();` so the iteration doesn't hold any borrow on `buffer`.
- **Files modified:** `src/color.rs`.
- **Verification:** `cargo test -p happyterminals-backend-ratatui --lib color` — all 33 color tests pass.
- **Committed in:** `43f9dfa` (Task 2 commit).

**3. [Rule 1 - Bug] Clippy lint churn in Task 1 palette (const casts, doc backticks)**
- **Found during:** Task 1 (first clippy run).
- **Issue:** `cast_sign_loss` on `(dr*dr+dg*dg+db*db) as u32`, `cast_possible_truncation` on `i as u8` in `nearest_256`, and `doc_markdown` on unbacked `ColorMode` in color.rs module docs.
- **Fix:** Per-site `#[allow(clippy::cast_sign_loss)]` on `sq_dist` (sum of squares is always non-negative), per-site `#[allow(clippy::cast_possible_truncation)]` on the single narrowing `best_i = i as u8` line (bounded by `i < 256` loop invariant), added backticks around `ColorMode` and `$COLORTERM`/`$TERM`.
- **Files modified:** `src/color.rs`, `src/color/palette.rs`.
- **Verification:** `cargo clippy -p happyterminals-backend-ratatui -- -D warnings` — clean.
- **Committed in:** `7817afb` (Task 1), refinements rolled into `43f9dfa` (Task 2) for color.rs.

**4. [Rule 1 - Bug] Clippy `match_same_arms` + `similar_names` cleanups in Task 2**
- **Found during:** Task 2 (full clippy pass).
- **Issue:** `Some("dumb") => ColorMode::Mono` duplicated `None => ColorMode::Mono` body; `Color::Reset => Color::Reset` arm in Mono branch duplicated the wildcard; `new_fg`/`new_bg` local binding names too similar.
- **Fix:** Merged TERM match arms to `Some("dumb") | None => ColorMode::Mono` with intent-comment; collapsed Mono `map_color` to `Color::Reset` single expression; renamed locals to `fg_out`/`bg_out`.
- **Files modified:** `src/color.rs`.
- **Verification:** `cargo clippy -p happyterminals-backend-ratatui -- -D warnings` clean; all cascade tests still pass.
- **Committed in:** `43f9dfa` (Task 2 commit).

**5. [Rule 1 - Bug] Doc warning — public `PALETTE_16_LUT` linked to private `SYSTEM_16`**
- **Found during:** Task 2 (cargo doc verification).
- **Issue:** `PALETTE_16_LUT` rustdoc used `[`SYSTEM_16`]` intra-doc link, but `SYSTEM_16` is `pub(crate)`. `cargo doc` emitted warnings about linking from a public item to a private one.
- **Fix:** Reworded doc to describe the concept ("xterm system-16 palette") without intra-doc link to the private item.
- **Files modified:** `src/color/palette.rs`.
- **Verification:** `cargo doc -p happyterminals-backend-ratatui --no-deps` clean (zero warnings).
- **Committed in:** `43f9dfa` (Task 2 commit).
