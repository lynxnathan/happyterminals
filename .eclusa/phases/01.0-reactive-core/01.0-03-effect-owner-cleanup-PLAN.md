## Decisions this plan executes

1. **Cycle detection = ImmediateEffect's panic-on-recursion.** We add NO custom panic wrapper. The runtime panics; the test in 01.0-05 catches-and-asserts `result.is_err()` only. Exact message is reactive_graph's internal string (CONTEXT.md §"Cycle detection" was amended in 01.0-01 to reflect this).

2. **`on_cleanup` out-of-scope behavior:**
   - **Debug** (`cfg(debug_assertions)`): `panic!` with the message `"happyterminals_core::on_cleanup called outside an owner scope"`. This is one of the three panics allowed by `#![deny(clippy::panic)]` — we use `std::panic::panic_any` or allow-attribute the single call site.
   - **Release**: no-op with `eprintln!("[happyterminals_core::on_cleanup] warning: called outside owner scope; callback dropped")`. **We do NOT add `tracing` as a dependency this plan** — CONTEXT.md §"Decisions" mentions `tracing::warn!` but also §"Dependencies to activate" does not list `tracing`. Research confirms `tracing` is not in the populated Cargo.toml. Decision: `eprintln!` for release; upgrade to `tracing` in a later phase when the backend wires a subscriber.

3. **`Owner::current()` API path** — use whichever path 01.0-01's spike validated. If `Owner::current()` exists in reactive_graph, use it. If not, use the `thread_local!` counter fallback that 01.0-01 installed in `runtime.rs`. Check `.eclusa/phases/01.0-reactive-core/01.0-01-SUMMARY.md` first.

4. **`Effect` does NOT expose a `.dispose()` method.** CONTEXT.md §"API surface" says "Effect is dropped when its Owner is disposed." Dropping the `Effect` value also drops the inner handle (auto-disposal per reactive_graph). We hold `_inner: ImmediateEffect` to keep the handle alive for the Effect's user-facing lifetime; it's auto-disposed on drop.

5. **`#[track_caller]` on constructors** — `Effect::new`, `create_root`, `on_cleanup`. Enables better panic messages in user code (the caller file:line appears in the panic rather than our internal location).

6. **Debug labels (`.label("name")`) — NOT in this plan.** CONTEXT.md §"Claude's Discretion" mentioned optional labeled cycle messages; the cycle-detection decision (01.0-01 amendment) deferred labeled messages to a future phase. No `Effect::with_label` / `Owner::with_label` in v0.0.0.

7. **Crate-root `pub use` re-exports are OUT OF SCOPE.** A separate Wave-3 plan (`01.0-02b-lib-reexports`) owns `lib.rs`. This plan's public items are reachable via their `pub mod effect;` / `pub mod owner;` paths (landed by 01.0-01) — e.g., `happyterminals_core::effect::Effect`. 02b promotes them to crate root.

8. **Smoke test lives in-file.** A `#[cfg(test)] mod smoke { ... }` at the bottom of `src/effect.rs` asserts that an Effect re-runs exactly twice (once at construction + once after a single `Signal::set`). This replaces the prior revision's ephemeral-example file dance — it is a real `#[test]`, run by `cargo test -p happyterminals-core --lib`, with an automated verify step that checks `test result: ok`. No files are created and deleted mid-plan.

    ```rust
    //! `Owner`, `create_root`, `on_cleanup` — the scope tree.

    use std::marker::PhantomData;
    use reactive_graph::owner::Owner as RgOwner;
    use send_wrapper::SendWrapper;  // or reactive_graph's re-export; match 01.0-01's resolved path

    /// A reactive scope handle. Disposing an Owner cascades cleanup to
    /// every Effect, Memo, and [`on_cleanup`] callback registered inside it.
    ///
    /// **Single-threaded.** `Owner` is `!Send + !Sync`.
    pub struct Owner {
        inner: RgOwner,
        _not_send: PhantomData&lt;*const ()&gt;,
    }

    impl Owner {
        /// Runs `f` with `self` set as the current reactive owner. Signals,
        /// memos, and effects created inside register under this owner.
        pub fn run_in&lt;R&gt;(&amp;self, f: impl FnOnce() -&gt; R) -&gt; R {
            self.inner.with(f)
        }

        /// Disposes this owner, running cleanup callbacks and dropping every
        /// Effect/Memo registered inside its scope.
        pub fn dispose(self) {
            self.inner.cleanup();
            // inner drops at end of scope.
        }
    }

    /// Creates a new root reactive scope, runs `f` inside it, and returns
    /// the value plus the owner handle. Drop or `.dispose()` the Owner to
    /// clean up everything allocated inside.
    #[track_caller]
    pub fn create_root&lt;R&gt;(f: impl FnOnce() -&gt; R) -&gt; (R, Owner) {
        let inner = RgOwner::new_root(None);
        let result = inner.with(f);
        (result, Owner { inner, _not_send: PhantomData })
    }

    /// Registers `f` to run when the current owner is disposed.
    ///
    /// # Panics (debug builds only)
    ///
    /// Panics if called outside an active owner scope. In release builds
    /// this logs a warning to stderr and silently drops `f`.
    #[track_caller]
    pub fn on_cleanup(f: impl FnOnce() + 'static) {
        if !owner_scope_active() {
            #[cfg(debug_assertions)]
            {
                // Allow-scope the one legal panic. This is the only place
                // in the crate where we panic intentionally.
                #[allow(clippy::panic)]
                { panic!("happyterminals_core::on_cleanup called outside an owner scope"); }
            }
            #[cfg(not(debug_assertions))]
            {
                eprintln!(
                    "[happyterminals_core::on_cleanup] warning: called outside owner scope; callback dropped"
                );
                return;
            }
        }

        // Wrap f to satisfy reactive_graph's Send+Sync+'static bound while
        // preserving our single-threaded !Send surface. FnOnce + SendWrapper:
        let wrapped = SendWrapper::new(f);
        reactive_graph::owner::on_cleanup(move || {
            // Unwrap the FnOnce from the SendWrapper; call it once.
            let inner = wrapped.take();  // take() moves out; SendWrapper supports it.
            inner();
        });
    }

    /// Scope-check helper. Returns true if a reactive Owner is currently
    /// active on this thread. Implementation switches between the direct
    /// `Owner::current()` API path and the thread_local fallback depending
    /// on what 01.0-01's spike validated; see `runtime.rs`.
    fn owner_scope_active() -&gt; bool {
        // Path A (preferred, if Owner::current exists):
        reactive_graph::owner::Owner::current().is_some()
        // Path B (fallback): use runtime::owner_scope_active() installed in 01.0-01.
        // Consult .eclusa/phases/01.0-reactive-core/01.0-01-SUMMARY.md for which path.
    }
    ```

If `take()` is absent in the version reactive_graph transitively pulls, use an alternative pattern: wrap `f` in an `Option<F>` inside the SendWrapper and `.take()` from the Option on first invocation.

    - **The `[allow(clippy::panic)]` attribute** is the only tolerated exception in the crate. Scope it as narrowly as possible (single expression), not file- or module-wide.

    - **`eprintln!` in release path** — documented as the v0.0.0 story. A later phase may upgrade to `tracing::warn!` when the backend pulls a subscriber.

    - **Path A/B selection** — hardcode to Path A for now.     mod smoke {
        //! In-file smoke test: Effect re-runs exactly twice when a dependency
        //! changes once. Validates REACT-03 wiring end-to-end. Lives here
        //! (not in tests/) because it is a small, deterministic, in-crate
        //! sanity check that MUST pass before plan 01.0-03 lands.

        use crate::effect::Effect;
        use crate::owner::create_root;
        use crate::signal::Signal;
        use std::cell::Cell;
        use std::rc::Rc;

        #[test]
        fn smoke_effect_reruns_twice_on_one_dep_change() {
            let runs = Rc::new(Cell::new(0u32));
            let runs_e = runs.clone();

            let (sig, owner) = create_root(|| {
                let s = Signal::new(0i32);
                let s_clone = s.clone();
                // Keep the Effect alive for the duration of the scope by
                // leaking its handle into a Box. Dropping the Effect in-scope
                // would disconnect the subscription before set() fires.
                let eff = Effect::new(move || {
                    // Read the signal to register the subscription.
                    let _ = s_clone.get();
                    runs_e.set(runs_e.get() + 1);
                });
                // Keep `eff` alive until the owner disposes it. We return it
                // alongside `s` so the outer scope holds both handles.
                (s, eff)
            });

            // Destructure: we want `s` so we can set it, but `eff` must stay alive.
            let (s, _eff) = sig;

            assert_eq!(
                runs.get(),
                1,
                "Effect did not run once at construction (got {})",
                runs.get()
            );

            // Single dependency change.
            s.set(42);

            assert_eq!(
                runs.get(),
                2,
                "Effect did not re-run exactly once after a single set() \
                 (expected 2 total runs, got {})",
                runs.get()
            );

            owner.dispose();
        }
    }
