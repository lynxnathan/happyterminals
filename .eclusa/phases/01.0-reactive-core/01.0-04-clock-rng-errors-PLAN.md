## Decisions this plan executes

1. **`Clock` trait is minimal:** a single `fn now(&self) -> Instant` method. No `from_unix_time` / `elapsed_since` helpers in v0.0.0. If future code needs more, extend the trait or compose.

2. **`Rng` trait is minimal:** `gen_u64(&mut self) -> u64` and `gen_f32(&mut self) -> f32`. CONTEXT.md §"API surface" locks this. No `RngCore` inheritance — downstream consumers don't pay a `rand` semver tax.

3. **`test-util` feature gate** — `ManualClock` and `SeededRng` live in `cfg(feature = "test-util")` modules nested inside `clock.rs` / `rng.rs` respectively. The feature was already added to `Cargo.toml` by 01.0-01 (`[features] test-util = []`). Integration tests in 01.0-05 enable the feature on the `happyterminals-core` dev-dep line (`features = ["test-util"]`).

4. **`ThreadRng` is `!Send` automatically** (because `rand::rngs::ThreadRng` is `!Send`). We don't need an explicit `PhantomData` marker — the field's non-Sendness propagates.

5. **`CoreError` is intentionally small.** One `NotInitialized` variant is enough to land the type; future fallible APIs extend the enum. `thiserror 2.0` derive handles `Display` + `Error`.

6. **`rand 0.9` API:** use `rand::rng()` (not `thread_rng()`), `rng.next_u64()` via `RngCore`, and `rng.random::<f32>()` (not `gen::<f32>()`). RESEARCH §"State of the Art" is explicit about this — don't regress.

7. **Preempt the `Rng as RandRng` unused-import clippy warning.** The `use rand::{rng, Rng as RandRng, RngCore};` line brings the `.random::<T>()` extension method into method-resolution scope but clippy's `unused_imports` lint may flag `RandRng` as unused (it is never named, only used via method call). Task 2's code skeleton applies `#[allow(unused_imports)]` directly above the `use` line with an explanatory comment. This saves an executor loop cycle.

8. **Crate-root `pub use` re-exports are OUT OF SCOPE.** A separate Wave-3 plan (`01.0-02b-lib-reexports`) owns `lib.rs`. This plan's types are reachable via their `pub mod` paths (landed by 01.0-01) — `happyterminals_core::clock::SystemClock`, etc. 02b promotes them to crate root.

    /// Injectable monotonic-clock abstraction.
    pub trait Clock: 'static {
        /// Returns the current moment, monotonic.
        fn now(&amp;self) -&gt; Instant;

    /// Production Clock — wraps [`std::time::Instant::now`].
    #[derive(Debug, Default, Clone, Copy)]

        use std::cell::Cell;
        use std::time::{Duration, Instant};

        /// A test-only clock. Time advances only via explicit [`advance`][Self::advance].
        pub struct ManualClock {
            now: Cell&lt;Instant&gt;,

        impl ManualClock {
            /// Starts the clock at `start`.
            pub fn new(start: Instant) -&gt; Self {
                Self { now: Cell::new(start) }

            /// Advances the clock by `d`. Idempotent with respect to ordering
            /// (repeated calls accumulate).
            pub fn advance(&amp;self, d: Duration) {
                self.now.set(self.now.get() + d);
            }
        }

        impl Clock for ManualClock {
            fn now(&amp;self) -&gt; Instant {
                self.now.get()
            }
        }
    }

    //! Injectable RNG abstraction.
    //!
    //! The minimal API (`gen_u64`, `gen_f32`) keeps downstream crates free
    //! of any `rand` semver dependency. Production uses [`ThreadRng`]
    //! (rand 0.9's thread-local generator); tests use
    //! `test_util::SeededRng` (rand's `SmallRng` seeded from a `u64`).

    // `Rng as RandRng` is needed for the `.random::&lt;T&gt;()` extension method to
    // resolve on `rand::rngs::ThreadRng` / `SmallRng`. Clippy's
    // `unused_imports` lint can false-flag it because the trait is never
    // named — only its methods are invoked. Do NOT remove `RandRng`.
    #[allow(unused_imports)]
    use rand::{rng, Rng as RandRng, RngCore};
    use rand::rngs::ThreadRng as RandThreadRng;

    /// Injectable RNG abstraction.
    pub trait Rng: 'static {
        /// Returns a uniform random `u64`.
        fn gen_u64(&amp;mut self) -&gt; u64;

        /// Returns a uniform random `f32` in `[0, 1)`.
        fn gen_f32(&amp;mut self) -&gt; f32;
    }

    /// Production RNG — wraps [`rand::rng`] (rand 0.9's thread-local generator).
    ///
    /// `!Send` (inherited from `rand::rngs::ThreadRng`).
    pub struct ThreadRng {
        inner: RandThreadRng,
    }

    impl Default for ThreadRng {
        fn default() -&gt; Self {
            Self { inner: rng() }
        }
    }

    impl Rng for ThreadRng {
        fn gen_u64(&amp;mut self) -&gt; u64 {
            self.inner.next_u64()
        }

        fn gen_f32(&amp;mut self) -&gt; f32 {
            self.inner.random::&lt;f32&gt;()
        }
    }

    #[cfg(feature = "test-util")]
    pub mod test_util {
        use super::Rng;
        // Same `unused_imports` preemption rationale as the parent module —
        // `Rng as RandRng` brings the `.random::&lt;T&gt;()` method into scope.
        #[allow(unused_imports)]
        use rand::{Rng as RandRng, RngCore, SeedableRng};
        use rand::rngs::SmallRng;

        /// Seeded deterministic RNG for snapshot-stable tests.
        pub struct SeededRng {
            inner: SmallRng,
        }

        impl SeededRng {
            /// Constructs a new `SeededRng` from a `u64` seed.
            pub fn new(seed: u64) -&gt; Self {
                Self { inner: SmallRng::seed_from_u64(seed) }
            }
        }

        impl Rng for SeededRng {
            fn gen_u64(&amp;mut self) -&gt; u64 { self.inner.next_u64() }
            fn gen_f32(&amp;mut self) -&gt; f32 { self.inner.random::&lt;f32&gt;() }
        }
    }
