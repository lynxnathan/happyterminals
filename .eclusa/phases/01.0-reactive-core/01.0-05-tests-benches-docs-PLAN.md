## Decisions this plan executes

1. **The cycle test asserts `result.is_err()` only** — not a specific message string. CONTEXT.md §"Cycle detection" was amended in 01.0-01; we honor that.

2. **`tests/transitions_10k.rs` is Linux-only** via `#![cfg(target_os = "linux")]`. Non-Linux CI ignores the file entirely; Linux CI runs it.

3. **The criterion bench has a hard `< 1µs` gate.** If it fails in CI, the phase fails. Implementer runs it locally before commit.

4. **`tests/send_sync.rs` is a compile-time test** — uses `static_assertions` dev-dep. This plan adds `static_assertions = "1.1"` to `[dev-dependencies]` in `Cargo.toml`. It's a tiny, stable, well-known crate.

5. **Docstring policy:** every public item in `src/` gets a `///` doc block that includes:
   - What the item does (one line).
   - Cross-thread behavior ("Single-threaded; !Send + !Sync" for all reactive types).
   - A short `# Examples` section where the API is non-obvious (Signal, Memo, Effect, batch — not needed on simple structs like `SystemClock`).

6. **No `insta` snapshots this phase.** CONTEXT.md §"Testing strategy" mentioned insta optional for cycle-panic message. Since we accepted reactive_graph's default message (decision 1 above, dated 01.0-01), there's nothing stable to snapshot. `insta` stays in dev-deps but is unused in this plan; a later phase may adopt it for more complex panic-format work.

7. **Tests that need `ManualClock` / `SeededRng`:** none in v1.0's integration suite. REACT-09's coverage is "types exist and implement trait correctly" — that's covered by `cargo check --features test-util` already. Adding a deterministic snapshot test using `SeededRng` is out of scope; surfaces in Phase 1.4 when scenes have time-dependent props.

    ```rust
    //! Criterion bench: Memo&lt;T&gt; equality-skip overhead.
    //!
    //! Gates: every `bench_function` must complete in &lt; 1µs per iteration.
    //! Enforced by CI: if criterion reports &gt; 1µs on the current machine,
    //! the build fails.

    use criterion::{black_box, criterion_group, criterion_main, Criterion};
    use happyterminals_core::{create_root, Memo, Signal};

    fn bench_memo_f32(c: &amp;mut Criterion) {
        c.bench_function("memo_f32_eq_skip", |b| {
            let (signals_and_memo, _owner) = create_root(|| {
                let s = Signal::new(0.0f32);
                let s_clone = s.clone();
                let m = Memo::new(move || s_clone.get() * 2.0);
                (s, m)
            });
            let (s, m) = signals_and_memo;

            b.iter(|| {
                s.set(black_box(0.0));
                black_box(m.get());
            });
        });
    }

    fn bench_memo_small_struct(c: &amp;mut Criterion) {
        #[derive(Clone, PartialEq)]
        struct Vec3 { x: f32, y: f32, z: f32 }

        c.bench_function("memo_vec3_eq_skip", |b| {
            let (stuff, _owner) = create_root(|| {
                let s = Signal::new(Vec3 { x: 0.0, y: 0.0, z: 0.0 });
                let s_clone = s.clone();
                let m = Memo::new(move || {
                    let v = s_clone.get();
                    Vec3 { x: v.x + 1.0, y: v.y, z: v.z }
                });
                (s, m)
            });
            let (s, m) = stuff;

            b.iter(|| {
                s.set(black_box(Vec3 { x: 0.0, y: 0.0, z: 0.0 }));
                black_box(m.get());
            });
        });
    }

    criterion_group!(benches, bench_memo_f32, bench_memo_small_struct);
    criterion_main!(benches);
    ```
