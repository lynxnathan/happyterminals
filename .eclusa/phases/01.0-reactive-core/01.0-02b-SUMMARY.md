## Decisions Made

- **Accepted rustfmt's line-alphabetical ordering** over the plan's hand-authored "alphabetical-by-module-in-CONTEXT-order" (signal → memo → batch → effect → owner → clock → rng → error). Rustfmt mandates `batch → clock → effect → error → memo → owner → rng → signal` and also reorders the inner braces (`owner::{Owner, create_root, on_cleanup}` instead of plan's `owner::{create_root, on_cleanup, Owner}`). Both orderings satisfy the plan's "stable diff across future phases" intent, and `cargo fmt --check` is a project-wide gate the plan explicitly requires to pass. Treating rustfmt as authoritative keeps the re-export block free of `#[rustfmt::skip]` hacks.
- **Test-util types (ManualClock, SeededRng) stay nested** under `::clock::test_util::` / `::rng::test_util::`. Consumers (01.0-05 test suite) will use the full path. Reaffirms the decision from 01.0-04 §context.8.
- **Did NOT add `#[doc(inline)]`** on the re-exports. The plan's `<open_questions>` §3 explicitly defers this to 01.0-05's doc-polish pass.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] Rustfmt reordered the re-export block**
- **Found during:** Task 2 (`cargo fmt --check` step)
- **Issue:** The plan's `<interfaces>` and `<action>` specified alphabetical-by-module order following CONTEXT.md (signal → memo → batch → effect → owner → clock → rng → error), but `cargo fmt --check` enforces line-alphabetical ordering for groups of `pub use` lines separated from other items by a blank line, and it also reorders types inside `{...}` braces. The initial commit attempt had fmt diff output and `cargo fmt --check` exited 1, which is a blocker for Task 2's success gate.
- **Fix:** Ran `cargo fmt` to apply rustfmt's canonical ordering. Resulting order is `batch, clock, effect, error, memo, owner, rng, signal` (line-alphabetical) and `owner::{Owner, create_root, on_cleanup}` (alphabetical inside braces). All eight `pub use` lines and every re-exported type remain present; only the line order and brace internal order changed.
- **Files modified:** `crates/happyterminals-core/src/lib.rs`
- **Verification:** `cargo fmt --check` exits 0. All 8 verify greps still match (grep patterns are order-independent per line). `cargo check --workspace` + both clippy flavors + `cargo doc` all pass. Every re-exported type remains reachable as `happyterminals_core::<Name>` (verified by inspection; 01.0-05's `use happyterminals_core::{...}` test imports will be the authoritative runtime proof).
- **Committed in:** `6e0aac7` (folded into the plan's single chore commit)
