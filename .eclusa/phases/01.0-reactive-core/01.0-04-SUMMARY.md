# Phase 01.0 Plan 04: Clock + Rng + CoreError Summary

**Landed the injectable-dependencies surface of the reactive core — `Clock` / `SystemClock` / `ManualClock`, `Rng` / `ThreadRng` / `SeededRng`, and the `thiserror`-derived `CoreError` — all on rand 0.9's current API (`rand::rng()`, `.random::<f32>()`) with a `test-util` feature gate for the deterministic test doubles.**

## What Shipped

### `clock.rs` (~65 lines)
- `pub trait Clock: 'static { fn now(&self) -> Instant; }` — minimal, one method.
- `SystemClock` — unit struct, `#[derive(Debug, Default, Clone, Copy)]`, wraps `Instant::now()`.
- `#[cfg(feature = "test-util")] pub mod test_util` containing `ManualClock` with `Cell<Instant>` interior mutability so `advance(&self, d)` works through `&dyn Clock`.

### `rng.rs` (~85 lines)
- `pub trait Rng: 'static { fn gen_u64; fn gen_f32; }` — minimal, two methods. No `RngCore` inheritance.
- `ThreadRng` — wraps `rand::rngs::ThreadRng`, default-constructed via `rand::rng()` (rand 0.9 API). `!Send` by inheritance.
- `#[cfg(feature = "test-util")] pub mod test_util` containing `SeededRng` wrapping `SmallRng::seed_from_u64`.
- `#[allow(unused_imports)]` applied inline above both `use rand::{…, Rng as RandRng, …};` lines (parent module + test_util submodule) to preempt clippy's `unused_imports` lint — `RandRng` is imported only for method-resolution of `.random::<T>()`, never named.

### `error.rs` (~30 lines)
- `#[derive(thiserror::Error, Debug)] pub enum CoreError` with a single `NotInitialized` variant (`#[allow(dead_code)]` + `#[error("…")]`).
- Compile-time assertion fn `_assert_core_error_send_sync` guards the Send+Sync contract for the future Python binding layer.

## Open-Questions Answers (from PLAN.md)

1. **Does `rand::rng()` exist?** Yes. rand 0.9 exposes `rand::rng()` as the non-deprecated thread-local constructor. No `#[allow(deprecated)]` needed.
2. **Does `SmallRng` need a feature flag?** No — default-enabled in rand 0.9. `use rand::rngs::SmallRng` works without touching `Cargo.toml`.
3. **Does `rand::rngs::ThreadRng` impl `Default`?** Not relevant — we implement `Default` on our own `ThreadRng` via `rng()` in the body, not via delegation.
4. **Is `NotInitialized` ever returned?** No — placeholder for future fallible API surface. `#[allow(dead_code)]` keeps the crate ready for a project-wide `deny(dead_code)` upgrade.
