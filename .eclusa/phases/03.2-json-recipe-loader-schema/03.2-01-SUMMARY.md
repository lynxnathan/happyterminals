# Phase 03.2 Plan 01: JSON Recipe Loader Summary

**JSON recipe loader with schemars-generated schema, jsonschema validation, and round-trip builder<->JSON property tests**

## Accomplishments
- `load_recipe()` parses, validates, and converts JSON recipes to SceneIr+CameraConfig
- `recipe_schema()` generates a full JSON Schema describing cameras, nodes, transforms, props
- jsonschema validation catches malformed input with path+description errors before deserialization
- 8 round-trip tests prove JSON recipes produce structurally identical SceneIr to the Rust builder
- All 3 camera variants (Orbit, FreeLook, FPS) correctly mapped
- Mesh nodes mapped to Custom("mesh") with path prop
- Test count increased from 394 to 415 (21 new tests, all passing)

## Decisions Made
- `load_recipe` returns `(SceneIr, CameraConfig)` tuple rather than `Scene` to avoid Scene validation at recipe load time -- the caller can wrap in Scene if needed
- RecipeError is kept in the DSL crate, not extending SceneError (per CONTEXT.md)
- JSON props stored as `PropValue::Static(Box<serde_json::Value>)` since JSON values lack typed Rust equivalents
- `instance_path()` is a method not a field in jsonschema 0.46 (fixed during compilation)
