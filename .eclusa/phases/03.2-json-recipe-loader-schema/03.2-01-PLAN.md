    **Step 2: Create `src/json.rs` with recipe types.** Per CONTEXT.md decisions:

    Root: `Recipe { version: String, camera: RecipeCamera, layers: Vec<RecipeLayer> }`. Use `#[serde(rename = "$version")]` for the version field. Derive `Serialize, Deserialize, JsonSchema` on all recipe structs.

    Camera (tagged enum via `#[serde(tag = "type")]`):
    - `RecipeCamera::Orbit { azimuth: f32, elevation: f32, distance: f32, target: [f32; 3] }`
    - `RecipeCamera::FreeLook { position: [f32; 3], yaw: f32, pitch: f32, speed: f32 }`
    - `RecipeCamera::Fps { position: [f32; 3], yaw: f32, pitch: f32, speed: f32, ground_y: f32 }`
    Use `#[serde(rename_all = "lowercase")]` on the enum.

    Layer: `RecipeLayer { z_order: i16, children: Vec<RecipeNode> }`.

    Node (tagged enum via `#[serde(tag = "type")]`):
    - `RecipeNode::Cube { transform: Option<RecipeTransform>, props: Option<HashMap<String, serde_json::Value>> }`
    - `RecipeNode::Mesh { path: String, transform: Option<RecipeTransform>, props: Option<HashMap<String, serde_json::Value>> }`
    - `RecipeNode::Group { children: Vec<RecipeNode>, transform: Option<RecipeTransform> }`
    Use `#[serde(rename_all = "lowercase")]` on the enum.

    Transform: `RecipeTransform { position: Option<[f32; 3]>, rotation: Option<[f32; 3]>, scale: Option<[f32; 3]> }`. All fields optional, defaulting to Transform::default() values.

    **Step 3: Implement `RecipeError`.** Enum with variants: `UnsupportedVersion { found: String }`, `SchemaValidation(String)`, `Deserialize(String)`. Implement `std::fmt::Display` and `std::error::Error`. Do NOT extend SceneError -- keep recipe errors in the DSL crate (per Claude's Discretion in CONTEXT.md).

    **Step 4: Implement `recipe_schema()`.** `pub fn recipe_schema() -> serde_json::Value { schemars::schema_for!(Recipe).to_value() }`. Verify schemars 1.x API -- it may be `SchemaGenerator::default().into_root_schema_for::<Recipe>()` or `schemars::generate::SchemaSettings::default().into_generator().into_root_schema_for::<Recipe>()`. Check schemars 1.2 docs. The schema must be self-describing enough for an LLM to author a valid recipe.

    **Step 5: Implement `load_recipe(json: &str) -> Result<SceneIr, RecipeError>`.** Steps inside:
    1. Parse JSON: `serde_json::from_str::<serde_json::Value>(json)` -- map err to RecipeError::Deserialize.
    2. Schema validate: `jsonschema::validate(&schema, &value)` -- collect errors into RecipeError::SchemaValidation with path + description.
    3. Check version: Deserialize into Recipe, check `recipe.version == "1.0"` -- return UnsupportedVersion if not.
    4. Convert Recipe -> SceneIr: Map each RecipeLayer to a SceneNode with NodeKind::Layer, map children recursively. For Cube: NodeKind::Cube. For Mesh: NodeKind::Custom("mesh".into()) with "path" in props as PropValue::Static(Box::new(path)). For Group: NodeKind::Group with recursive children. Map RecipeTransform to Transform (use defaults for None fields). Map props HashMap to PropMap with PropValue::Static(Box::new(json_value)) for each entry -- store as serde_json::Value in the Box since JSON values don't have typed Rust equivalents.
    5. Return `Ok(SceneIr::new(layer_nodes))`.

    **Step 6: Implement camera conversion.** `fn recipe_camera_to_config(cam: RecipeCamera) -> CameraConfig` mapping each variant's fields to the corresponding camera struct. Use `Vec3::from_array()` for [f32; 3] -> Vec3 conversions.

    **Step 7: Register module.** Add `pub mod json;` to `src/lib.rs`.

    **Note on SceneIr:** The `SceneIr` comparison in round-trip tests is structural, not pointer-identity (per CONTEXT.md). SceneIr does NOT derive PartialEq -- the test must walk the tree and compare kind, transform fields, and props manually. Add a helper `fn scene_ir_structurally_equal(a: &SceneIr, b: &SceneIr) -> bool` in the test module.
