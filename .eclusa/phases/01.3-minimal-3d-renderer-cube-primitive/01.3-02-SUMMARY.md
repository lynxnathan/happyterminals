## Decisions Made
- Compensate Y-flip winding reversal by checking edge function sign against total area sign -- CCW triangles become CW after projection Y-flip, so both windings must be accepted
- Scene-fit near plane: z_near = (camera.distance - 0.866).max(0.01) instead of using Projection's fixed 0.1, for better depth precision
- Added ratatui-core as direct renderer dependency for Position type in grid write-back
- Used char::encode_utf8 with stack-allocated [u8; 4] buffer for zero-alloc set_symbol calls

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Fixed Y-flip winding reversal in rasterizer**
- **Found during:** Task 1 (draw_renders_visible_cube test)
- **Issue:** After screen-space projection flips Y, CCW-wound triangles become CW, causing all edge functions to be negative. The rasterizer only accepted positive edge functions, producing zero visible output.
- **Fix:** Check the sign of the total triangle area and accept edge functions matching that sign (all >= 0 for CCW, all <= 0 for CW)
- **Files modified:** crates/happyterminals-renderer/src/rasterizer.rs
- **Verification:** draw_renders_visible_cube test passes, cube is visible
- **Committed in:** 60c9100 (Task 1 commit)
