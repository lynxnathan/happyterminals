# Phase 02.5 Plan 02: Resize Hardening Summary

**Skip-frame resize_pending flag added to run(), run_with_input(), and run_scene() preventing stale z-buffer renders during rapid terminal resize**

## Accomplishments
- All three run variants (run, run_with_input, run_scene) hardened with resize_pending flag
- Resize events set the flag; next tick skips render callback and only resizes grid + writes blank buffer
- run_scene additionally skips tick_fn when resize is pending to avoid scene state updates with stale viewport
- 13 occurrences of resize_pending across the file (3 declarations, 3 set-true, 3 check-true, 3 reset-false, 1 guard for tick_fn)

## Decisions Made
- Skip render callback entirely on resize tick rather than attempting a partial render -- the 16-33ms gap is imperceptible and prevents stale z-buffer artifacts
- In run_scene, tick_fn is also skipped during resize since scene state updates with stale viewport dimensions could produce incorrect projection math
