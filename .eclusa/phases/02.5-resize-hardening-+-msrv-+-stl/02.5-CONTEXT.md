<decisions>
## Implementation Decisions

### Resize-Race Hardening
- **Strategy:** Drain + clamp + re-rasterize. When `Event::Resize(w, h)` arrives, drain any queued events, clamp grid dimensions to new size, and let the next tick re-render at the new size. No mid-frame resize ‚Äî the rasterizer only sees stable dimensions.
- **Location:** `crates/happyterminals-backend-ratatui/src/run.rs` ‚Äî the event loop already handles `Event::Resize` via `InputEvent::Resize`. Harden by adding a resize flag that skips the current frame's render if dimensions changed mid-tick.
- **Test:** Synthetic stress test ‚Äî rapid-fire `Event::Resize` events in a loop, assert no panics and grid dimensions always match terminal size after settling.

### STL Loader
- **Dep:** `stl_io 0.11` ‚Äî lightweight, stable, no heavy deps. Add to `happyterminals-renderer` as an optional feature or direct dep.
- **API:** `pub fn load_stl(path: impl AsRef<Path>) -> Result<(Mesh, LoadStats), MeshError>` ‚Äî same signature pattern as `load_obj`. Returns the same `Mesh` struct. STL files only have triangles (no quads/ngons), so no triangulation step needed.
- **Normals:** STL files include per-facet normals. Use them directly (no flat-normal recomputation unless the stored normal is degenerate).
- **Error surface:** Extend `MeshError` with an `StlParse` variant. Same `thiserror` derive pattern.
- **Fixtures:** Add 2-3 STL test fixtures (simple cube, small mesh). Reuse the OBJ corpus test pattern from Phase 2.1.

### MSRV Pin
- **Version:** Rust 1.86 (current stable as of project start).
- **File:** `rust-toolchain.toml` at workspace root with `channel = "1.86"`.
- **CI:** Add a matrix job that builds with MSRV. Use `cargo-msrv verify` or `cargo +1.86 build --workspace`.
- **Enforcement:** Any use of a post-1.86 feature fails CI.

### Claude's Discretion
- Whether `stl_io` is an optional feature flag or always-on dep in the renderer crate.
- Exact resize drain strategy (loop until no more events vs single drain pass).
- CI workflow file details (GitHub Actions YAML structure).
- Whether to add STL models to the model-viewer example or keep a separate STL example.

</decisions>

## Specific Ideas

- The resize stress test should fire 100+ resize events in rapid succession and verify the grid settles to the final size with no panics.
- STL loader can share the degenerate-triangle skip logic from the OBJ loader (cross product magnitude check).
- Model-viewer could be extended to also cycle through STL files if any are present, but this is optional ‚Äî a simple `load_stl` test is sufficient for the requirement.

</specifics>
