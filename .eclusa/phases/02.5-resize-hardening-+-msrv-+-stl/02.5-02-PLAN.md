<objective>
Harden the resize path in run.rs to prevent rendering with stale dimensions.

Purpose: Windows Terminal (and other terminals) can fire resize events mid-tick. The current code resizes the grid on the next tick via frame.area(), but the Renderer's z-buffer may be sized for old dimensions. Adding a skip-frame flag ensures the grid is resized and the next tick renders cleanly.
Output: All three run variants in run.rs hardened with resize_pending flag.
</objective>

    Per CONTEXT decision: drain + clamp + re-rasterize with skip-frame strategy.

    For each of the three `run` variants (`run`, `run_with_input`, `run_scene`):

    1. Add `let mut resize_pending = false;` before the main loop.

    2. In the **Resize event handler** (the `InputEvent::Resize { width, height }` match arm), add:
       ```rust
       resize_pending = true;
       ```
       Keep the existing `input_signals.terminal_size.set((*width, *height))` call.

    3. In the **tick arm**, inside `terminal.draw(|frame| { ... })`, after `grid.resize(frame.area())`:
       - If `resize_pending` is true:
         - Set `resize_pending = false;`
         - Still call `grid.resize(frame.area())` to update dimensions.
         - Clone the grid buffer, apply downsample, write to frame -- but **skip the render callback** (do NOT call `render_fn` / `walk_and_render` / `tick_fn` for this frame).
         - Use `return;` to exit the draw closure early after writing the blank/resized buffer.
       - If `resize_pending` is false: execute the normal render path (unchanged).

    Concrete changes per variant:

    **`run()` (lines 78-89 tick arm):**
    ```rust
    _ = tick.tick() => {
        let _span = tracing::trace_span!("frame").entered();
        terminal.draw(|frame| {
            grid.resize(frame.area());
            if resize_pending {
                resize_pending = false;
                let mut out = grid.deref().clone();
                downsample(&mut out, color_mode);
                *frame.buffer_mut() = out;
                return;
            }
            render_fn(&mut grid, &input_signals);
            let mut out = grid.deref().clone();
            downsample(&mut out, color_mode);
            *frame.buffer_mut() = out;
        })?;
    }
    ```

    **`run_with_input()` (lines 172-187 tick arm):**
    Same pattern -- skip `render_fn(&mut grid, &input_signals, &input_map)` when resize_pending. Still call `input_map.tick_update(dt)` and `input_map.reset_axes()` after the draw closure (those are outside the closure).

    **`run_scene()` (lines 294-317 tick arm):**
    Same pattern -- skip `tick_fn(dt, &input_signals)` AND `walk_and_render(...)` AND pipeline when resize_pending. The `tick_fn` call is before `terminal.draw()`, so wrap it:
    ```rust
    _ = tick.tick() => {
        let _span = tracing::trace_span!("frame").entered();
        if !resize_pending {
            tick_fn(dt, &input_signals);
        }
        terminal.draw(|frame| {
            grid.resize(frame.area());
            if resize_pending {
                resize_pending = false;
                let mut out = grid.deref().clone();
                downsample(&mut out, color_mode);
                *frame.buffer_mut() = out;
                return;
            }
            // ... normal render path unchanged
        })?;
    }
    ```

    4. The `resize_pending` variable must be declared as `let mut resize_pending = false;` and captured mutably by the `loop { tokio::select! { ... } }` block. Since `resize_pending` is set in the event arm and read in the tick arm, both within the same loop body (not across async boundaries), this is straightforward -- no `Arc` or `Cell` needed.
