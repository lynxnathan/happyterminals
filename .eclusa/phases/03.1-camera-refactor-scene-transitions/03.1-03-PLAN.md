  <action>
    1. Add a new `pub async fn run_scenes` function in run.rs. Per RESEARCH recommendation: separate function (not extending run_scene) to avoid bloating its signature.

       Signature:
       ```rust
       pub async fn run_scenes<F>(
           transition_manager: TransitionManager,
           spec: FrameSpec,
           input_map: InputMap,
           mut tick_fn: F,
       ) -> Result<(), Box<dyn std::error::Error>>
       where
           F: FnMut(Duration, &InputSignals, &mut TransitionManager),
       ```

       The tick_fn receives a mutable ref to TransitionManager so the user can trigger transitions programmatically from within the callback.

    2. The frame loop follows the existing `run` pattern (tokio::select between tick interval and EventStream), but adds:
       - Call `input_map.process_event()` for input events (same pattern as run() but using InputMap).
       - On each frame tick:
         a. Call `tick_fn(dt, &input_signals, &mut transition_manager)` -- user logic.
         b. Call `transition_manager.tick(dt)` -- advance transition progress.
         c. If `transition_manager.is_transitioning()`:
            - Render current "from" scene into `buf_a` (clone the Grid's buffer, render, extract).
            - Render "to" scene into `buf_b`.
            - Call `transition_manager.blend(&buf_a, &buf_b, &mut output_buf)`.
            - Write output_buf to the frame.
         d. If not transitioning:
            - Render current scene normally (same as run_scene).
         e. Apply color downsampling, flush.

    3. For the two-buffer rendering during transition:
       - Use two `Renderer` instances (per RESEARCH Pitfall 5: shared z-buffer interference).
       - Create them once before the loop, not per-frame.
       - Each scene gets its own Grid clone for rendering.

    4. Handle resize during transition: if `resize_pending` is true while transitioning, skip blend for that frame (render blank), re-render both scenes at new size on next frame. Per RESEARCH Pitfall 3.

    5. Export `run_scenes` from `crates/happyterminals-backend-ratatui/src/lib.rs`.

    6. Run `cargo test -p happyterminals-backend-ratatui && cargo clippy -p happyterminals-backend-ratatui -- -D warnings`.

    NOTE: The rendering of individual scenes within run_scenes should reuse the existing `walk_and_render` / `render_node` logic from run.rs. The scene's SceneGraph is walked the same way; the difference is we do it twice (once per buffer) during transitions.
  </action>
