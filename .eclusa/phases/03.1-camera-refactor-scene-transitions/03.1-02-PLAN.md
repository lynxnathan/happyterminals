       - Per CONTEXT.md locked decision on trait signature.

       - `pub struct Dissolve;` impl: iterate all cells; if `progress < 0.5` copy from `a`, else copy from `b`. Use `ratatui_core::layout::Position::new(x, y)` and `cell(pos)` / `cell_mut(pos)`.

       - `pub struct SlideLeft;` impl: compute `split_x = area.x + (width * (1.0 - progress)) as u16`. Cells with `x < split_x` copy from `a`, else from `b`.

       - `pub struct FadeToBlack;` impl: two-phase. If `progress < 0.5`: copy from `a` but at sub-progress `p = progress / 0.5`, replace cells with spaces when `p > threshold` (use column-based threshold: `(x as f32 / width) < p` means space). If `progress >= 0.5`: copy from `b` but at sub-progress `p = (progress - 0.5) / 0.5`, show cells when `(x as f32 / width) < p`, else space. This creates a left-to-right wipe through black.

       - For each effect, when copying a cell: use `out_cell.set_symbol(src_cell.symbol())` and `out_cell.set_style(src_cell.style())`.

       - Unit tests creating small 4x2 Buffers with known cell content, verifying blend output at progress 0.0, 0.5, 1.0.

    1. Replace the scaffold in `transition.rs` with the full implementation. Keep the module doc-comment updated.

    2. Define `TransitionState` enum (per RESEARCH recommendation):
       ```rust
       pub enum TransitionState {
           Idle { scene: Scene, owner: Owner },
           Transitioning {
               from_scene: Scene,
               from_owner: Owner,
               to_scene: Scene,
               to_owner: Owner,
               effect_name: String,
               elapsed: Duration,
               duration: Duration,
               easing: fn(f32) -> f32,
           },
       }
       ```

    3. Rewrite `TransitionManager`:
       - Fields: `state: Option<TransitionState>`, `registry: HashMap<String, Box<dyn TransitionEffect>>`
       - `new()` -> registers 3 built-in effects ("dissolve" -> Dissolve, "slide-left" -> SlideLeft, "fade-to-black" -> FadeToBlack), state = None.
       - `set_scene(scene, owner)` -> if existing Idle state, dispose old owner. Set state = Idle { scene, owner }.
       - `register(name: impl Into<String>, effect: impl TransitionEffect + 'static)` -> inserts into registry.
       - `transition_to(scene: Scene, owner: Owner, effect_name: &str, duration: Duration) -> Result<(), SceneError>` -> validates effect_name exists in registry, moves from Idle to Transitioning with easing=linear (default). Return Err if not in Idle state or effect not found.
       - `transition_to_with_easing(scene, owner, effect_name, duration, easing: fn(f32) -> f32)` -> same but with custom easing.
       - `tick(dt: Duration)` -> if Transitioning: add dt to elapsed; compute `progress = easing(elapsed.as_secs_f32() / duration.as_secs_f32()).clamp(0.0, 1.0)`. If progress >= 1.0: dispose `from_owner`, move to Idle with `to_scene`/`to_owner`.
       - `blend(&self, buf_a: &Buffer, buf_b: &Buffer, output: &mut Buffer) -> bool` -> if Transitioning: look up effect by name in registry, call `effect.blend(buf_a, buf_b, progress, output)`, return true. If Idle: return false.
       - `is_transitioning(&self) -> bool`
       - `current_scene(&self) -> Option<&Scene>` -> returns scene from current state (Idle or to_scene in Transitioning).

    4. Use `crate::error::SceneError` for error returns (or add a variant if needed).

    5. Update lib.rs re-exports: `pub use transition::{TransitionManager, TransitionState};`

    CRITICAL INVARIANTS (per CONTEXT.md locked decisions):
    - Owner disposal happens ONLY when progress >= 1.0 (transition complete), synchronously between frames.
    - During transition, BOTH owners exist (both scenes are live).
    - `Owner::dispose(self)` takes ownership, preventing double-dispose.
