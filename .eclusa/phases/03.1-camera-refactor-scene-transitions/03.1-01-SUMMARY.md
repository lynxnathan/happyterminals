# Phase 3.1 Plan 01: Camera Polymorphism Summary

**Renderer::draw() and draw_particles() accept &dyn Camera with eye/forward extracted from view matrix inverse; OrbitCamera shim removed from render_node**

## Accomplishments
- Renderer::draw() and draw_particles() now accept `&dyn Camera` instead of `&OrbitCamera`
- Eye position and camera direction extracted from view matrix inverse (universal for any camera type)
- scene_fit_near generalized from `&OrbitCamera` to `&Mat4`, computing Euclidean distance to mesh center
- OrbitCamera shim removed from render_node in run.rs; replaced with CameraConfig::as_camera()
- 368 workspace tests pass (365 existing + 3 new polymorphism tests), all examples compile clean

## Task Commits

_Note: Task 1 used TDD flow (RED: failing tests, GREEN: implementation)_

## Files Created/Modified
- `crates/happyterminals-renderer/src/lib.rs` - draw()/draw_particles() signatures changed to &dyn Camera, scene_fit_near generalized to &Mat4, eye/forward extracted from view matrix inverse
- `crates/happyterminals-scene/src/camera.rs` - Added CameraConfig::as_camera() returning &dyn Camera
- `crates/happyterminals-backend-ratatui/src/run.rs` - render_node now uses camera_config.as_camera() directly, shim removed

## Decisions Made
- Eye position extracted from view.inverse().col(3).truncate() rather than adding eye()/forward() to the Camera trait -- view matrix IS the universal representation per RESEARCH guidance
- Camera forward direction is the negated Z-axis column of the view inverse (-inv.col(2)), correct for RH coordinate system
- scene_fit_near now computes Euclidean distance from eye to mesh bounding sphere center, replacing OrbitCamera::distance field access
